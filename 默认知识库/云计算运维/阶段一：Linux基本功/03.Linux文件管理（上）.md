# <font style="color:rgb(51, 51, 51);">二、文件命名规则</font>
## <font style="color:rgb(51, 51, 51);">可以使用哪些字符？</font>
<font style="color:rgb(51, 51, 51);">理论上除了字符“/”之外，所有的字符都可以使用，但是要注意，在目录名或文件名中，不建议使用某些特殊字符，例如， <、>、？、* 等，尽量避免使用。</font>

<font style="color:rgb(51, 51, 51);">工作时文件的命名规则：</font>

<font style="color:rgb(51, 51, 51);">① 由于linux严格区分大小写，所以尽量都用小写字母</font>

<font style="color:rgb(51, 51, 51);">② 如果必须对文件名进行分割，建议使用"_"，例如：lhp_bj_2020.log</font>

> <font style="color:rgb(119, 119, 119);">_ 下划线，在Linux操作系统中，可以使用Shift键 + 减号</font>
>

<font style="color:rgb(51, 51, 51);">a.txt</font>

<font style="color:rgb(51, 51, 51);">001.txt</font>

<font style="color:rgb(51, 51, 51);">002.txt</font>

<font style="color:rgb(51, 51, 51);">tongxunlu.txt</font>

<font style="color:rgb(51, 51, 51);">tongxunlu_bj_caiwu.txt</font>

## <font style="color:rgb(51, 51, 51);">文件名的长度</font>
<font style="color:rgb(51, 51, 51);">目录名或文件名的长度不能超过 255 个字符</font>

> <font style="color:rgb(119, 119, 119);">尽量不要太长，另外文件名称一定要见名知意，可以使用英文单词</font>
>

## <font style="color:rgb(51, 51, 51);">文件名的大小写</font>
<font style="color:rgb(51, 51, 51);">Linux目录名或文件名是区分大小写的。如 lhp、LHP、yunwei 和 Yunwei ，是互不相同的目录名或文件名。</font>

<font style="color:rgb(51, 51, 51);">个人建议：</font>

<font style="color:rgb(51, 51, 51);">不要使用字符大小写来区分不同的文件或目录。建议文件名一律使用小写字母</font>

## <font style="color:rgb(51, 51, 51);">Linux文件扩展名</font>
<font style="color:rgb(51, 51, 51);">Linux文件的扩展名对 Linux 操作系统没有特殊的含义，Linux 系统并不以文件的扩展名开分区文件类型。例如，lhp.exe 只是一个文件，其扩展名 .exe 并不代表此文件就一定是可执行的。</font>

<font style="color:rgb(51, 51, 51);">在Linux系统中，文件扩展名的用途为了使运维人员更好的区分不同的文件类型。</font>

> <font style="color:rgb(119, 119, 119);">在Linux操作系统中，文件的类型是依靠权限位的标识符来进行区分的。当然也可以通过颜色，如黑色普通文件，天蓝色文件夹</font>
>

# <font style="color:rgb(51, 51, 51);">三、文件管理命令</font>
<font style="color:rgb(51, 51, 51);">在日常工作中，我们经常需要对Linux的文件或目录进行操作，常见操作包括新建，删除，更改，查看，复制，移动等。</font>

## <font style="color:rgb(51, 51, 51);">目录创建</font>
<font style="color:rgb(51, 51, 51);">在实际应用中，与目录相关的操作主要有两个：创建目录与删除目录</font>

### <font style="color:rgb(51, 51, 51);">mkdir创建目录</font>
<font style="color:rgb(51, 51, 51);">基本语法：</font>

```shell
# mkdir 目录名称
```

<font style="color:rgb(51, 51, 51);">主要功能：就是根据目录的名称创建一个目录</font>

> <font style="color:rgb(119, 119, 119);">mkdir = make directory</font>
>

<font style="color:rgb(51, 51, 51);">案例：在家目录下创建一个lhp的文件夹</font>

```shell
# mkdir lhp
```

### <font style="color:rgb(51, 51, 51);">mkdir -p递归创建目录</font>
<font style="color:rgb(51, 51, 51);">基本语法：</font>

```shell
# mkdir -p /usr/local/nginx/conf
选项说明：
-p：递归创建，从左边的路径开始一级一级创建目录，直到路径结束
```

<font style="color:rgb(51, 51, 51);">案例：在已知目录（/usr/local）下创建多级目录（nginx/conf）</font>

```shell
# mkdir /usr/local/nginx/conf
mkdir: cannot create directory ‘/usr/local/nginx/conf’: No such file or directory
mkdir：不能创建目录conf，为什么呢？因为/usr/local目录下并没有nginx
```

<font style="color:rgb(51, 51, 51);">解决方案：如果我们想同时创建多级目录，必须使用-p选项，其会从左向右一级一级创建目录。</font>

```shell
# mkdir -p /usr/local/nginx/conf
```

### <font style="color:rgb(51, 51, 51);">使用mkdir同时创建多个目录</font>
```shell
# mkdir 目录名称1 目录名称2 目录名称3 ...
```

<font style="color:rgb(51, 51, 51);">案例：在当前目录下创建a、b、c三个目录</font>

```shell
# mkdir a
# mkdir b
# mkdir c
或
# mkdir a b c
```

### 总结
<font style="color:rgb(51, 51, 51);">一个目录一个目录创建# mkdir 目录名称</font>

<font style="color:rgb(51, 51, 51);">多级目录一起创建# mkdir -p 目录名称1/目录名称2/...</font>

<font style="color:rgb(51, 51, 51);">同时创建多个目录# mkdir 目录名称1 目录名称2 目录名称3</font>

> <font style="color:rgb(51, 51, 51);">在终端中：</font>
>
> <font style="color:rgb(51, 51, 51);">按方向键=>向上，其会返回上一次输入的命令</font>
>
> <font style="color:rgb(51, 51, 51);">按方向键=>向下，其会返回下一次输入的命令</font>
>

## <font style="color:rgb(51, 51, 51);">目录删除（必须是空目录）</font>
### <font style="color:rgb(51, 51, 51);">rmdir移除空目录</font>
<font style="color:rgb(51, 51, 51);">基本语法：</font>

```shell
# rmdir 目录名称
```

> <font style="color:rgb(119, 119, 119);">rmdir = remove directory，移除目录</font>
>

<font style="color:rgb(51, 51, 51);">案例：把家目录下的lhp删除（空目录）</font>

```shell
# rmdir lhp
```

### <font style="color:rgb(51, 51, 51);">rmdir -p递归删除空目录</font>
<font style="color:rgb(51, 51, 51);">基本语法：</font>

```shell
# rmdir -p 目录名称1/目录名称2/目录名称3/...
```

<font style="color:rgb(51, 51, 51);">主要功能：从右向左一级一级删除空目录</font>

<font style="color:rgb(51, 51, 51);">案例：递归删除lhp文件夹中的yunwei文件夹中的shenzhen3</font>

```shell
# rmdir -p lhp/yunwei/shenzhen3
```

### <font style="color:rgb(51, 51, 51);">使用rmdir同时删除多个空目录</font>
```shell
# rmdir 目录名称1 目录名称2 目录名称3 ...
```

<font style="color:rgb(51, 51, 51);">案例：删除家目录中的a、b、c三个空目录</font>

```shell
# rmdir a b c
```

### 总结
<font style="color:rgb(51, 51, 51);">#</font><font style="color:rgb(51, 51, 51);"> rmdir 目录名称，删除某个空目录</font>

<font style="color:rgb(51, 51, 51);">#</font><font style="color:rgb(51, 51, 51);"> rmdir -p 目录名称1/目录名称2/目录名称3/...，同时删除多级目录（从右向左一级一级删除）</font>

<font style="color:rgb(51, 51, 51);"># rmdir 目录名称1 目录名称2 目录名称3，同时删除多个空目录</font>

## <font style="color:rgb(51, 51, 51);">文件创建</font>
### <font style="color:rgb(51, 51, 51);">touch创建文件</font>
<font style="color:rgb(51, 51, 51);">基本语法：</font>

```shell
# touch 文件名称
```

<font style="color:rgb(51, 51, 51);">主要功能：在Linux系统中的当前目录下创建一个文件</font>

<font style="color:rgb(51, 51, 51);">案例：在当前目录下创建一个readme.txt文件</font>

```shell
# touch readme.txt
```

### <font style="color:rgb(51, 51, 51);">使用touch命令同时创建多个文件</font>
<font style="color:rgb(51, 51, 51);">基本语法：</font>

```shell
# touch 文件名称1 文件名称2 文件名称3 ...
```

<font style="color:rgb(51, 51, 51);">案例：创建一个shop商城文件夹，然后在内部创建index.php/admin.php/config.php三个文件</font>

```shell
# mkdir shop
# touch shop/index.php
# touch shop/admin.php
# touch shop/config.php
或
# touch shop/index.php shop/admin.php shop/config.php
```

> <font style="color:rgb(119, 119, 119);">提出问题：观察以下命令，看看有何区别？</font>
>
> <font style="color:rgb(119, 119, 119);">第一种写法：touch shop/index.php shop/admin.php shop/config.php</font>
>
> <font style="color:rgb(119, 119, 119);">代表在shop目录下首先创建一个index.php，再创建一个admin.php，最后config.php</font>
>
> <font style="color:rgb(119, 119, 119);"></font>
>
> <font style="color:rgb(119, 119, 119);">第二种写法：touch shop/index.php admin.php config.php</font>
>
> <font style="color:rgb(119, 119, 119);">代表在shop目录下首先创建index.php，然后在当前目录下创建admin.php，最后在当前目录下创建config.php</font>
>

### <font style="color:rgb(51, 51, 51);">根据序号同时创建多个文件</font>
<font style="color:rgb(51, 51, 51);">基本语法：</font>

```shell
# touch 文件名称{开始序号..结束序号}
含义：
根据提供的开始序号~结束序号，一个一个创建文件
```

<font style="color:rgb(51, 51, 51);">案例：创建file1、file2、file3、file4、file5共5个文件</font>

```shell
# touch file1
# touch file2
# touch file3
# touch file4
# touch file5
```

<font style="color:rgb(51, 51, 51);">或</font>

```shell
# touch file{1..5}
```

### 总结
<font style="color:rgb(51, 51, 51);">#</font><font style="color:rgb(51, 51, 51);"> touch 文件名称</font>

<font style="color:rgb(51, 51, 51);">#</font><font style="color:rgb(51, 51, 51);"> touch 文件名称1 文件名称2 文件名称3 ...</font>

<font style="color:rgb(51, 51, 51);"># touch 文件名称{开始序号..结束序号}</font>

## <font style="color:rgb(51, 51, 51);">文件删除</font>
### <font style="color:rgb(51, 51, 51);">普通删除</font>
<font style="color:rgb(51, 51, 51);">基本语法：</font>

```shell
# rm  [选项]  文件或文件夹的名称
选项说明：
-r ：针对文件夹，代表递归删除，先把目录下的所有文件删除，然后再删除文件夹
-f ：强制删除，不提示，初学者一定要慎重！！！
```

<font style="color:rgb(51, 51, 51);">案例：删除readme.txt文件</font>

```shell
# rm readme.txt
rm: remove regular empty file ‘readme.txt’?这里可以回复y or n
y代表确认删除
n代表取消删除
```

<font style="color:rgb(51, 51, 51);">案例：强制删除admin.php，不提示（慎重）</font>

```shell
# rm -f admin.php
```

<font style="color:rgb(51, 51, 51);">案例：删除非空的文件夹（-rf强制递归删除，不提示）</font>

```shell
# rm -rf shop
```

<font style="color:rgb(51, 51, 51);">提示信息：descend into directory 'itheima/'?</font><font style="color:rgb(51, 51, 51);">含义：是否进入itheima目录？</font>

<font style="color:rgb(51, 51, 51);">提示信息：remvoe directory 'itheima/'?含义：是否删除itheima文件夹？</font>

<font style="color:rgb(51, 51, 51);">扩展：Ctrl + C在Linux下的使用方式</font>

> <font style="color:rgb(119, 119, 119);">在Linux操作系统中，Ctrl + C不是复制，而是中断某个操作</font>
>

**<font style="color:rgb(51, 51, 51);">总结：</font>**

<font style="color:rgb(51, 51, 51);">#</font><font style="color:rgb(51, 51, 51);"> rm 文件名称</font>

<font style="color:rgb(51, 51, 51);">#</font><font style="color:rgb(51, 51, 51);"> rm 文件名称1 文件名称2 文件名称3 ... </font>

<font style="color:rgb(51, 51, 51);">#</font><font style="color:rgb(51, 51, 51);"> rm -f 文件名称 => 强制删除，不提示</font>

<font style="color:rgb(51, 51, 51);"># rm -rf 文件夹名称 		=> 递归删除文件夹</font>

### <font style="color:rgb(51, 51, 51);">扩展：rm -rf /*，一定要慎重</font>
```shell
# rm -rf /*	    =>   代表删除根目录下的所有文件（如果没有备份，恢复的几率不高）
rm ：删除
-r ：递归删除（无论文件夹是否为空）
-f ：强制删除不提示
/  ：代表根分区
*  ：所有
```

# cp [选项] 源文件或文件夹 目标路径

## <font style="color:rgb(51, 51, 51);">tar打包压缩与解压缩</font>
### <font style="color:rgb(51, 51, 51);">几个概念</font>
**<font style="color:rgb(51, 51, 51);">打包</font>**<font style="color:rgb(51, 51, 51);">：默认情况下，Linux的压缩概念一次只能压缩一个文件。针对多文件或文件夹无法进行直接压缩。所以需要提前对多个文件或文件夹进行打包，这样才可以进行压缩操作。</font>

<font style="color:rgb(51, 51, 51);">1.txt 5MB</font>

<font style="color:rgb(51, 51, 51);">2.txt 10MB</font>

<font style="color:rgb(51, 51, 51);">3.txt 15MB</font>

<font style="color:rgb(51, 51, 51);">1.txt + 2.txt + 3.txt = 打包 = 123.tar = 30MB</font>

> <font style="color:rgb(119, 119, 119);">打包只是把多个文件或文件夹打包放在一个文件中，但是并没有进行压缩，所以其大小还是原来所有文件的总和。</font>
>

**<font style="color:rgb(51, 51, 51);">压缩</font>**<font style="color:rgb(51, 51, 51);">：也是一个文件和目录的集合，且这个集合也被存储在一个文件中，但它们的不同之处在于，压缩文件所占用的磁盘空间</font>**<font style="color:rgb(51, 51, 51);">比集合中所有文件大小的总和要小。</font>**

<font style="color:rgb(51, 51, 51);">1.txt 5MB</font>

<font style="color:rgb(51, 51, 51);">2.txt 10MB</font>

<font style="color:rgb(51, 51, 51);">3.txt 15MB</font>

<font style="color:rgb(51, 51, 51);">1.txt + 2.txt + 3.txt = 压缩 = 123.tar.gz = 20MB（体积变小了）</font>

### <font style="color:rgb(51, 51, 51);">tar打包操作</font>
<font style="color:rgb(51, 51, 51);">基本语法：</font>

```shell
# tar [选项]  打包后的名称.tar  多个文件或文件夹
选项说明：
-c ：打包
-f ：filename，打包后的文件名称
-v ：显示打包的进度
-u ：update缩写，更新原打包文件中的文件（了解）
-t ：查看打包的文件内容（了解）
```

<font style="color:rgb(51, 51, 51);">案例：把a.txt、b.txt、c.txt文件打包到abc.tar文件中</font>

```shell
# tar -cvf abc.tar a.txt b.txt c.txt
```

<font style="color:rgb(51, 51, 51);">案例：把wechat文件夹进行打包wechat.tar</font>

```shell
# tar -cvf wechat.tar wechat
```

### <font style="color:rgb(51, 51, 51);">tar -tf以及tar -uf</font>
```shell
# tar -tf 打包后的文件名称
```

<font style="color:rgb(51, 51, 51);">主要功能：查看tar包中的文件信息</font>

<font style="color:rgb(51, 51, 51);">案例：查看abc.tar包中的文件信息</font>

```shell
# tar -tf abc.tar
```



```shell
# tar -uf 打包后的文件名称
```

> <font style="color:rgb(119, 119, 119);">u = update，更新，如果还想往tar包中更新或追加内容都可以通过-u选项</font>
>

<font style="color:rgb(51, 51, 51);">案例：向abc.tar包中添加一个d.txt文件</font>

```shell
# touch d.txt
# tar -uf abc.tar d.txt
查看是否添加成功
# tar -tf abc.tar
```

<font style="color:rgb(51, 51, 51);"></font>

<font style="color:rgb(51, 51, 51);">扩展：如何把tar包中的文件释放出来</font>

```shell
打包
# tar -cf  abc.tar  a.txt  b.txt  c.txt

打包 => 释放（-c 变成 -x）

释放
# tar -xf  abc.tar
```

### <font style="color:rgb(51, 51, 51);">打包并压缩（学习的重点）</font>
<font style="color:rgb(51, 51, 51);">基本语法：</font>

```shell
# tar [选项]  压缩后的压缩包名称  要压缩的文件或文件夹
选项说明：
-cf ：对文件或文件夹进行打包

-v  ：显示压缩进度

-z	：使用gzip压缩工具把打包后的文件压缩为.gz
-j  ：使用bzip2压缩工具把打包后的文件压缩为.bz2
-J	：使用xz压缩工具把打包后的文件压缩为.xz

压缩速度：gzip > bzip2 > xz
压缩率：gzip < bzip2 < xz
100M   90M     80M	  70M
```

<font style="color:rgb(51, 51, 51);">案例：把a.txt、b.txt、c.txt文件打包并压缩为abc.tar.gz</font>

```shell
# tar -zcf abc.tar.gz a.txt b.txt c.txt
```

<font style="color:rgb(51, 51, 51);">案例：把wechat文件夹压缩为wechat.tar.gz格式的压缩包</font>

```shell
# tar -zcf wechat.tar.gz wechat
```

### <font style="color:rgb(51, 51, 51);">对压缩包进行解压</font>
<font style="color:rgb(51, 51, 51);">解压过程非常简单，就是把压缩的参数中的c换成x就可以实现解压缩了</font>

```shell
*.tar.gz格式的压缩包
# tar -zxf 名称.tar.gz

*.tar.bz2格式的压缩包
# tar -jxf 名称.tar.bz2

*.tar.xz格式的压缩包
# tar -Jxf 名称.tar.xz
```

<font style="color:rgb(51, 51, 51);">案例：把abc.tar.gz格式的压缩包进行解压缩操作</font>

```shell
# tar -zxf abc.tar.gz
```

<font style="color:rgb(51, 51, 51);">案例：把wechat.tar.gz格式的压缩包进行解压缩操作</font>

```shell
# tar -zxf wechat.tar.gz
```

## <font style="color:rgb(51, 51, 51);">zip压缩与解压缩</font>
### <font style="color:rgb(51, 51, 51);">zip压缩</font>
<font style="color:rgb(51, 51, 51);">基本语法：</font>

```shell
# zip [选项] 压缩后的文件名称.zip  文件或文件夹
选项说明：
-r ：递归压缩，主要针对的是文件夹
```

> <font style="color:rgb(119, 119, 119);">Linux下已经有gzip、bzip2以及xz压缩命令了，为什么还需要使用zip压缩呢？</font>
>
> <font style="color:rgb(119, 119, 119);">答：zip格式在Windows以及Linux中都是可以正常使用的。</font>
>

<font style="color:rgb(51, 51, 51);">案例：把a.txt、b.txt、c.txt进行压缩为abc.zip</font>

```shell
# zip abc.zip a.txt b.txt c.txt
```

<font style="color:rgb(51, 51, 51);">案例：把wechat文件夹压缩为wechat.zip</font>

```shell
# zip -r wechat.zip wechat
```

### <font style="color:rgb(51, 51, 51);">unzip解压缩</font>
<font style="color:rgb(51, 51, 51);">基本语法：</font>

```shell
# unzip 压缩包名称		=>     解压到当前目录
# unzip 压缩包名称 -d 指定路径		=>   解压到指定路径下
```

<font style="color:rgb(51, 51, 51);">案例：对abc.zip文件进行解压缩</font>

```shell
# unzip abc.zip
```

<font style="color:rgb(51, 51, 51);">案例：把wechat.zip解压到/usr/local/nginx目录下</font>

```shell
# unzip wechat.zip -d /usr/local/nginx/
```

## 总结
<font style="color:rgb(51, 51, 51);">在Linux操作系统中，其拥有很多压缩工具，如gzip、bzip2、xz等等，但是其有一个缺点：</font>

<font style="color:rgb(51, 51, 51);">一次只能压缩一个文件（而且无法压缩文件夹）</font>

<font style="color:rgb(51, 51, 51);">① 打包，把一个或多个文件打成tar包，并不是压缩，只是把文件合并在一个tar文件中</font>

<font style="color:rgb(51, 51, 51);">1.txt 5MB</font>

<font style="color:rgb(51, 51, 51);">2.txt 10MB</font>

<font style="color:rgb(51, 51, 51);">3.txt 15MB</font>

<font style="color:rgb(51, 51, 51);">打包123.tar = 5MB + 10MB + 15MB = 30MB</font>

<font style="color:rgb(51, 51, 51);">为什么要打包，就是为了后期的压缩操作。</font>

<font style="color:rgb(51, 51, 51);">② 打包并压缩</font>

```shell
# tar -zcf 压缩包名称.tar.gz 文件或文件夹名称
# tar -jcf 压缩包名称.tar.bz2 文件或文件夹名称
# tar -Jcf 压缩包名称.tar.xz 文件或文件夹名称
```

<font style="color:rgb(51, 51, 51);">③ 解压缩</font>

```shell
# tar 把c参数更换为x，就可以实现解压缩  压缩包名称.tar.gz
```

<font style="color:rgb(51, 51, 51);">扩展：</font>

```shell
# tar -zxf 压缩包名称.tar.gz [参数] 指定路径
```

<font style="color:rgb(51, 51, 51);">第一种解决思路：随用随查，百度、Google</font>

```shell
# tar -zxf 压缩包名称.tar.gz -C 指定路径
```

<font style="color:rgb(51, 51, 51);">第二种解决思路：可以使用man命令（manual缩写，手册）</font>

```shell
# man tar命令

# tar -zxf 压缩包名称.tar.gz -C 指定路径
```

<font style="color:rgb(51, 51, 51);">④ zip压缩与解压缩</font>

<font style="color:rgb(51, 51, 51);">给我们提供一种既可以在Windows中使用的，也可以提供一种在Linux中使用的通用的压缩格式。</font>

```shell
# zip [-r]  压缩包名称.zip  要压缩的文件或文件夹
```

<font style="color:rgb(51, 51, 51);">解压缩：</font>

```shell
# unzip 压缩包名称.zip  -d 指定路径
```
