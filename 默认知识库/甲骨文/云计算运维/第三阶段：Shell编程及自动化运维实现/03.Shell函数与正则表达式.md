# <font style="color:rgb(51, 51, 51);">一、Shell函数</font>

<font style="color:rgb(51, 51, 51);">作用：Shell函数就是用于解决代码重用问题（让代码可以重复使用）</font>

## <font style="color:rgb(51, 51, 51);">什么是函数？</font>

<font style="color:rgb(51, 51, 51);">Shell中允许将一组命令集合或语句形成一段可用代码，这些代码块称为Shell函数。给这段代码起个名字称为函数名，后续可以直接调用该段代码的功能。</font>

## <font style="color:rgb(51, 51, 51);">函数的基本语法</font>

```shell
函数名()
{
  		函数体（一堆命令的集合，来实现某个功能）   
}

function 函数名()
{
   		函数体（一堆命令的集合，来实现某个功能）  
}


function_name() {
		command
		command
}


function function_name() {
		command
		command
}
```

<font style="color:rgb(51, 51, 51);">函数调用</font>

```shell
函数名称
```

<font style="color:rgb(51, 51, 51);">案例：</font>

```shell
menu(){
        cat <<-EOF
            h	显示命令帮助
            f	显示磁盘分区
            d	显示磁盘挂载
            m	查看内存使用
            u	查看系统负载
            q	退出程序
            EOF

}
menu		//调用函数
```

<font style="color:rgb(51, 51, 51);">小结：</font>

<font style="color:rgb(51, 51, 51);">函数有何作用？答：简化Shell脚本，让代码进行重用操作</font>

<font style="color:rgb(51, 51, 51);">注意：函数在定义时没有真正执行，只有在调用时才真正执行！</font>

## <font style="color:rgb(51, 51, 51);">函数的return返回值</font>

<font style="color:rgb(51, 51, 51);">作用：让函数执行完毕后，返回一个结果给</font><code><font style="color:rgb(51, 51, 51);">$?</font></code>

<font style="color:rgb(51, 51, 51);">基本语法：</font>

```shell
# 定义函数
func() {
    return 0-256中的任意值
}
# 调用函数
func

return可以结束一个函数，类似于前面讲的循环控制语句break
return可以返回0-256中的任意一个值，当然0就代表成功，非0代表失败
如果没有return命令，函数将返回最后一个Shell的退出值
```

<font style="color:rgb(51, 51, 51);">案例：</font>

```shell
#!/bin/bash

# 定义一个函数，计算两个数字的除法
divide() {
    if [ $2 -eq 0 ]; then
        echo "错误：除数不能为0"
        return 1  # 如果除数为0，返回 1，表示错误
    else
        result=$(($1 / $2))  # 执行除法运算
        echo "结果: $result"
        return 0  # 返回 0，表示成功
    fi
}

# 调用函数，传入参数
divide 10 2
echo "函数返回值: $?"  # 0表示成功

divide 10 0
echo "函数返回值: $?"  # 1表示除数为0的错误
```

<font style="color:rgb(51, 51, 51);">小结：</font>

<font style="color:rgb(51, 51, 51);">return可以结束一个函数，类似于前面讲的循环控制语句break(结束当前循环，执行循环体后面的代码)</font>

<font style="color:rgb(51, 51, 51);">return默认返回函数中最后一个命令的退出状态，也可以给定参数值，该参数值的范围是0-256之间。</font>

<font style="color:rgb(51, 51, 51);">如果没有return命令，函数将返回最后一个Shell的退出值。</font>

## <font style="color:rgb(51, 51, 51);">案例：定期检查磁盘使用率并告警</font>

<font style="color:rgb(51, 51, 51);">某公司需要确保其生产环境中的服务器磁盘空间不会耗尽，以免影响系统的正常运行。运维团队希望使用 Shell 脚本定期检查服务器磁盘使用情况，当某个磁盘分区的使用率超过设定的阈值时，自动发出告警信息（例如通过发送邮件或日志记录），以便运维团队及时处理。</font>

<font style="color:rgb(51, 51, 51);">项目需求：</font>

* <font style="color:rgb(51, 51, 51);">定期检查服务器上各个分区的磁盘使用情况。</font>
* <font style="color:rgb(51, 51, 51);">设定一个使用率的阈值（如 80%），当某个分区的使用率超过此阈值时发出告警。</font>
* <font style="color:rgb(51, 51, 51);">在告警时记录日志，并发送邮件给运维团队，提醒需要清理磁盘。</font>
* <font style="color:rgb(51, 51, 51);">使用 Shell 函数实现各项功能，包括检查磁盘、记录日志和发送告警。check\_disk\_usage()、alert\_disk\_usage()</font>

<font style="color:rgb(51, 51, 51);"></font>

<font style="color:rgb(51, 51, 51);">解决方案：使用 Shell 函数进行磁盘检查和告警</font>

<font style="color:rgb(51, 51, 51);">Shell 脚本思路：</font>

1. <font style="color:rgb(51, 51, 51);">编写一个 Shell 函数来检查所有分区的磁盘使用率。</font>
2. <font style="color:rgb(51, 51, 51);">编写另一个函数来记录告警信息并发送邮件通知。</font>
3. <font style="color:rgb(51, 51, 51);">将这个脚本设为定时任务，通过 </font><code><font style="color:rgb(51, 51, 51);background-color:rgb(243, 244, 244);">crontab</font></code><font style="color:rgb(51, 51, 51);"> 每隔一段时间执行。</font>

<font style="color:rgb(51, 51, 51);">编写脚本</font>

```shell
#!/bin/bash

# 定义告警日志文件和使用率阈值
LOGFILE="/var/log/disk_usage.log"
# 测试阈值可以小一些，生产建议80-85
THRESHOLD=5

# 定义告警邮箱
ALERT_EMAIL="15201008359@163.com"
FROM_EMAIL="centos9@server.com"

# 检查磁盘使用率的函数
check_disk_usage() {
    # 使用 df 命令获取每个分区的使用率
    df -h | grep '^/dev/' | while read line; do
        # 提取磁盘分区和使用率
        PARTITION=$(echo $line | cut -d' ' -f1)
        USAGE=$(echo $line | cut -d' ' -f5 | tr -d '%')
        

        # 如果使用率超过设定的阈值，调用告警函数
        if [ $USAGE -ge $THRESHOLD ]; then
            alert_disk_usage $PARTITION $USAGE
        fi
    done
}

# 告警函数，记录日志并发送邮件
alert_disk_usage() {
    PARTITION=$1
    USAGE=$2
    SUBJECT="磁盘使用率告警: $PARTITION 达到 $USAGE%"

    # 记录告警日志
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] 警告: 分区 $PARTITION 使用率已达到 $USAGE%" >> $LOGFILE

    # 发送告警邮件
    {
        echo "To: $ALERT_EMAIL"
        echo "From: $FROM_EMAIL"
        echo "Subject: $SUBJECT"
        echo "Content-Type: text/plain; charset=UTF-8"
        echo "Content-Transfer-Encoding: 8bit"
        echo ""
        echo "警告："
        echo "服务器上的分区 $PARTITION 使用率已达到 $USAGE%。请尽快检查并清理磁盘，避免服务中断。"
    } | sendmail -t
}

# 调用检查磁盘使用率的函数
check_disk_usage

注：read在读取磁盘信息时，会自动去除多余的空格
```

### <font style="color:rgb(51, 51, 51);">查什么（元字符）</font>

| **<font style="color:rgb(51, 51, 51);">代码</font>** | **<font style="color:rgb(51, 51, 51);">功能</font>** |
| :--- | :--- |
| <font style="color:rgb(51, 51, 51);">.</font> | <font style="color:rgb(51, 51, 51);">匹配任意1个字符（除了\n）</font> |
| <font style="color:rgb(51, 51, 51);">\[ ]</font> | <font style="color:rgb(51, 51, 51);">匹配\[ ]中列举的字符，字符簇，本身表示一个范围区间，每次只能匹配其中的某个字符</font> |
| <font style="color:rgb(51, 51, 51);">\[^指定字符]</font> | <font style="color:rgb(51, 51, 51);">匹配除了指定字符以外的所有字符，^托字节在字符簇中出现代表取反的意思</font> |
| <font style="color:rgb(51, 51, 51);">\d</font> | <font style="color:rgb(51, 51, 51);">匹配数字，即0-9，\d都称之为Perl风格语法</font> |

| <font style="color:rgb(51, 51, 51);">\s</font> | <font style="color:rgb(51, 51, 51);">匹配空白，即 空格，tab键</font> |

| <font style="color:rgb(51, 51, 51);">\w</font> | <font style="color:rgb(51, 51, 51);">匹配非特殊字符，即a-z、A-Z、0-9、\_</font> |
| <font style="color:rgb(51, 51, 51);">\W</font> | <font style="color:rgb(51, 51, 51);">匹配特殊字符，即非字母、非数字</font> |

> \[a-z]匹配26个小写字符中的任意一个，\[A-Z]匹配26个大写字符中的任意一个，\[0-9]任意一个数字，\[3-9]匹配3-9之间的任意一个数字，\[aeiou]匹配小aeiou五个字符中的任意一个

## <font style="color:rgb(51, 51, 51);">分组、捕获、反向引用</font>

<font style="color:rgb(51, 51, 51);">作用：主要是进行重复数据的匹配操作</font>

<font style="color:rgb(51, 51, 51);">举个栗子：1234-1111-1221-3333</font>

<font style="color:rgb(51, 51, 51);">要求：要求使用正则匹配1111以及3333这种格式的数字</font>

<font style="color:rgb(51, 51, 51);">要求：要求使用正则匹配1221这种格式的数字</font>

### <font style="color:rgb(51, 51, 51);">分组</font>

<font style="color:rgb(51, 51, 51);">在正则表达式中，通过一对圆括号括起来的内容，我们就称之为"分组"。</font>

<font style="color:rgb(51, 51, 51);">\d(\d)(\d)</font>

<font style="color:rgb(51, 51, 51);">正则表达式中</font><code><font style="color:rgb(51, 51, 51);">\d(\d)(\d)</font></code><font style="color:rgb(51, 51, 51);">中，</font><code><font style="color:rgb(51, 51, 51);">(\d)(\d)</font></code><font style="color:rgb(51, 51, 51);">就是子表达式，一共有两个</font><code><font style="color:rgb(51, 51, 51);background-color:rgb(243, 244, 244);">()</font></code><font style="color:rgb(51, 51, 51);">圆括号，则代表两个分组</font>

### <font style="color:rgb(51, 51, 51);">捕获</font>

<font style="color:rgb(51, 51, 51);">当正则表达式在字符串中匹配到相应的内容后，计算机系统会自动把分组所匹配的到内容放入到系统的对应缓存区中（缓存区从1开始）</font>

### <font style="color:rgb(51, 51, 51);">反向引用</font>

<font style="color:rgb(51, 51, 51);">在正则表达式中，我们可以通过\n（n代表第n个缓存区的编号）来引用缓存区中的内容，我们把这个过程就称之为"反向引用"。</font>

<font style="color:rgb(51, 51, 51);"></font>

# <font style="color:rgb(51, 51, 51);">三、expect</font>

<code><font style="color:rgb(51, 51, 51);background-color:rgb(243, 244, 244);">expect</font></code><font style="color:rgb(51, 51, 51);"> 是一个用于自动化交互的工具，通常用于自动化 SSH 登录、安装脚本、密码输入等需要用户交互的任务。以下是 </font><code><font style="color:rgb(51, 51, 51);background-color:rgb(243, 244, 244);">expect</font></code><font style="color:rgb(51, 51, 51);"> 的基础教程，包含基本概念、安装和常见用法。</font>

<font style="color:rgb(51, 51, 51);">作用：解决大部分脚本需要用户交互的场景，通过expect让脚本变成完全自动化形式！</font>

## <font style="color:rgb(51, 51, 51);">expect工具安装</font>

<font style="color:rgb(51, 51, 51);">安装expect</font>

```shell
# CentOS/RHEL
sudo dnf install expect -y
```

<font style="color:rgb(51, 51, 51);">expect 脚本一般包含以下几个基本元素：</font>

* **<font style="color:rgb(51, 51, 51);">spawn</font>**<font style="color:rgb(51, 51, 51);">：用于启动需要自动化的命令或程序。</font>
* **<font style="color:rgb(51, 51, 51);">expect</font>**<font style="color:rgb(51, 51, 51);">：定义程序期望看到的输出，expect 会等待输出内容符合条件时再进行下一步。</font>
* **<font style="color:rgb(51, 51, 51);">send</font>**<font style="color:rgb(51, 51, 51);">：向程序发送字符串，如输入命令或确认。</font>
* **<font style="color:rgb(51, 51, 51);">interact</font>**<font style="color:rgb(51, 51, 51);">：允许用户和脚本的交互，通常在脚本末尾使用，以保持会话打开。</font>

## <font style="color:rgb(51, 51, 51);">expect脚本结构</font>

```shell
#!/usr/bin/expect
# 设置变量
set timeout 10
set password "123456"

# 启动命令
spawn ssh root@192.168.88.102

# 匹配期望的输出
expect {
  "yes/no" {
    send "yes\r"
    exp_continue
  }
  "password:" {
    send "$password\r"
  }
}

# 保持会话交互
interact


以上内容解析：
#!/usr/bin/expect：指定脚本解释器。
set：用于设置变量，timeout 指定超时时间，超过时间后 expect 将退出。
spawn：启动一个命令，比如 ssh。
expect：等待命令的输出符合指定模式，类似于 “等待提示”。
send：向命令发送输入，$password\r 表示发送密码，并按下回车。
interact：保持交互模式，让用户可以手动操作（通常用于保持会话）。
```

```shell

# 捕获相关内容

    # 处理主机指纹确认

        send "yes\r"
        exp_continue
    }
    # 处理密码输入
    "password:" {
        send "123456\r"
    }
}

# 交互模式
interact
```
