## <font style="color:rgb(51, 51, 51);">zabbix 基础概念初探</font>

**<font style="color:rgb(51, 51, 51);">主机（host）和主机群组（host group）</font>**

<font style="color:rgb(51, 51, 51);">主机指被监控的一个设备(服务器，交换机等)，当被监控的主机数量巨大时，就需要分组</font>

***

**<font style="color:rgb(51, 51, 51);">zabbix 用户（user）与用户群组（group）</font>**

<font style="color:rgb(51, 51, 51);">zabbix 可以多个用户登录管理(和 Linux 操作系统一样有管理员和普通管理者)</font>

***

**<font style="color:rgb(51, 51, 51);">监控项（item）与应用集（application）</font>**

<font style="color:rgb(51, 51, 51);">监控的需求太多了，就拿监控 cpu 平均负载来说，就有监控1分钟内，5分钟内，15分钟内等三个常见的监控参数。</font>

<font style="color:rgb(51, 51, 51);">监控项（item）是从收集数据或监控的一个</font>**<font style="color:rgb(51, 51, 51);">最小单位</font>**<font style="color:rgb(51, 51, 51);">。把 cpu 1分钟内的平均负载就可以做成一个监控项。</font>

<font style="color:rgb(51, 51, 51);">应用集就是多个监控项的组。</font>

***

**<font style="color:rgb(51, 51, 51);">图形</font>**

<font style="color:rgb(51, 51, 51);">监控项收集的数据需要用图形直观地展示出来。</font>

***

**<font style="color:rgb(51, 51, 51);">触发器和报警</font>**

<font style="color:rgb(51, 51, 51);">当监控项收集的数据达到一个临界点时，就要触发报警通知管理人员。</font>

<font style="color:rgb(51, 51, 51);">如: 当根分区使用率超过 80% 时, 就通过发报警信息到管理人员。</font>

***

**<font style="color:rgb(51, 51, 51);">模板</font>**

<font style="color:rgb(51, 51, 51);">模板主要包括监控项，图形，触发器等概念，相当于是把要监控的东西做成一个合集。</font>

## <font style="color:rgb(51, 51, 51);">MySQL8 部署</font>

在 server 服务器安装 MySQL 数据库，因为 Zabbix 拉取到的数据要存储到 MySQL 数据库中！

<font style="color:rgb(51, 51, 51);">第一步：下载 MySQL</font>

```powershell
# wget https://dev.mysql.com/get/Downloads/MySQL-8.0/mysql-8.0.37-1.el9.x86_64.rpm-bundle.tar
```

<font style="color:rgb(51, 51, 51);">网速比较慢，也可以直接</font>**<font style="color:rgb(51, 51, 51);">使用资料中的 MySQL 离线安装包</font>**<font style="color:rgb(51, 51, 51);">直接安装！</font>

<font style="color:rgb(51, 51, 51);">第二步：解压 MySQL</font>

```powershell
# mkdir /root/mysql
# tar -xf mysql-8.0.37-1.el9.x86_64.rpm-bundle.tar -C /root/mysql

x：代表 "extract"，意思是提取归档文件中的内容。
f：告诉 tar 命令，接下来的参数是一个文件名，而不是要归档或提取的文件列表。
-C（注意大写）：这是一个选项，允许您指定在何处（即哪个目录）解压缩文件。
```

<font style="color:rgb(51, 51, 51);">第三步：安装 MySQL</font>

```powershell
# cd /root/mysql
# rpm -ivh mysql-community-* --force --nodeps

-i: 安装软件包。
-v: 显示详细输出。
-h: 显示安装进度。
--force: 强制覆盖已经存在的文件。这可能会破坏系统，因为它会覆盖现有的文件而不进行任何检查。
--nodeps: 安装软件包时，不检查其依赖关系。这可能会导致软件包无法正常工作，因为它可能缺少必要的库或依赖项。
```

<font style="color:rgb(51, 51, 51);">第四步：启动 MySQL，并开机启动</font>

```powershell
# 启动MySQL
# systemctl start mysqld
 
# MySQL加入开启自启
# systemctl enable mysqld
```

<font style="color:rgb(51, 51, 51);">第五步：查看初始密码</font>

```powershell
# 查询初始密码
# cat /var/log/mysqld.log | grep password
0U&dHcboug!%
```

<font style="color:rgb(51, 51, 51);">第六步：修改密码</font>

<font style="color:rgb(51, 51, 51);">如果你想要设置简单的密码，你需要先更换一个初始密码，然后再修改策略，再修改简单密码，所以先使用一个复杂的密码作为新密码。【注意：数据库中输入的指令注意“；”这个符合别漏掉】</font>

```powershell
# mysql -uroot -p
Enter password:输入初始密码
 
# 使用root登入mysql，密码使用使用上方查询出来的密码→修改密码为Xa%^3T%T4!
mysql> ALTER USER 'root'@'localhost' IDENTIFIED BY 'Xa%^3T%T4!';
 
# 设置密码检查等级
mysql> set global validate_password.policy=0;
 
# 0 (或LOW)：允许密码包含最简单的密码（只包含小写字母、数字、特殊字符中的一类）。
# 1 (或MEDIUM)：要求密码至少包含小写字母、大写字母、数字、特殊字符中的三类。
# 2 (或STRONG)：要求密码至少包含小写字母、大写字母、数字、特殊字符中的四类。
 
# 密码的最短长度
mysql> set global validate_password.length=5;
 
# 密码至少要包含的小写字母个数和大写字母个数
mysql> set global validate_password.mixed_case_count=0;
 
# 再次修改密码为qwe!123，并退出数据库
mysql> ALTER USER 'root'@'localhost' IDENTIFIED BY 'qwe!123';
 
# 退出
mysql> exit
```

<font style="color:rgb(51, 51, 51);">第二步：安装 PHP8 和其相关的组件</font>

```shell
# 下面是Zabbix server在使用PHP时需要用到的PHP相关组件的安装：
# sudo dnf install -y php80 php80-php-cli php80-php-gd php80-php-json php80-php-mbstring php80-php-mysqlnd php80-php-xml

php80：
这是PHP 8.0的主要软件包，包含了PHP的核心解释器和基础功能。

php80-php-cli：
PHP的命令行接口（CLI），允许你从命令行运行PHP脚本。这在开发、测试和维护PHP应用程序时非常有用。

php80-php-gd：
#GD库是一个用于处理图像的开源代码库，它提供了一系列函数来动态创建和操作图像。在Zabbix中，它可能被用于图形显示。

php80-php-json：
提供了对JSON格式的解析和生成支持。JSON是Web服务中常用的数据交换格式。

php80-php-mbstring：
提供了对多字节字符串的支持，包括UTF-8等字符集。对于国际化应用非常重要。

php80-php-mysqlnd：
MySQL的原生驱动（Native Driver），用于与MySQL数据库进行交互。由于Zabbix通常使用MySQL作为其后端数据库，因此这个扩展是必须的。

php80-php-xml：
提供了对XML的支持，包括解析和生成XML文档。虽然Zabbix不直接依赖于XML处理，但某些PHP扩展或应用程序可能需要它。

php80-php-bcmath：
BC Math库提供了高精度数学计算的支持。虽然Zabbix本身可能不需要它，但某些PHP应用程序或扩展可能需要它。

php80-php-fpm：
FastCGI进程管理器（FPM）是一个用于管理PHP FastCGI进程的工具。它允许你将PHP作为独立的进程运行，而不是作为Apache或Nginx的模块。如果你打算使用Nginx作为Web服务器，并希望PHP作为FastCGI进程运行，那么这个扩展是必须的。
```

## <font style="color:rgb(51, 51, 51);">安装 Zabbix</font>

Zabbix 底层是基于 PHP + MySQL 的，所以我们要有 LNMP 或者 LAMP 的环境才行！上面已经安装了 MySQL 和 PHP，而 **Zabbix 软件内部集成了 Apache 服务器**，所以不需要单独安装 Web 服务器了！

在 server 服务器中做如下操作。

<font style="color:rgb(51, 51, 51);">第一步：加载 zabbix 软件库（其实就是安装 Zabbix 官方提供的 yum 仓库）</font>

## <font style="color:rgb(51, 51, 51);">配置 Zabbix 数据库</font>

<font style="color:rgb(51, 51, 51);">第一步：创建数据库</font>

```shell
#进入数据库
# mysql -uroot -p
Enter password:输入之前设置root访问数据库的密码 => qwe!123
 
#创建了一个名为zabbix_proxy的新数据库，该数据库使用utf8mb4字符集和utf8mb4_bin校对规则，这使得它特别适合于存储包含特殊字符（如emoji）的数据，并且以二进制方式比较和排序字符串。
mysql> create database zabbix character set utf8mb4 collate utf8mb4_bin;

utf8mb4：
是一个Unicode字符集，可以存储多达4字节的字符。这个字符集特别有用，因为它可以存储像emoji这样的特殊字符，而标准的utf8字符集（实际上是utf8mb3）则不能。

utf8mb4_bin:
utf8mb4_bin是一个二进制校对规则，它基于字符的二进制值进行比较。这意味着它区分大小写，并且不会考虑字符的排序规则（如字母表顺序）。如果你想要一个不区分大小写的校对规则，你可能会选择utf8mb4_general_ci或utf8mb4_unicode_ci等。
```

<font style="color:rgb(51, 51, 51);">第二步：设置密码并授权</font>

```shell
#创建一个数据库访问账号zabbix，密码为qwe!123
mysql> create user zabbix@localhost identified by 'qwe!123';
 
#授予用户 zabbix 在 zabbix 数据库上所有权限，并且这个权限仅适用于从 localhost 主机连接到 MySQL 服务器的用户。
mysql> grant all privileges on zabbix.* to zabbix@localhost;

授予权限：用户 zabbix 将获得在 zabbix 数据库上执行任何操作的能力，包括创建表、删除表、插入数据、更新数据、查询数据等。

指定数据库：这里的 zabbix.* 表示 zabbix 数据库中的所有对象（表、视图、存储过程等）。

指定用户：zabbix@localhost 表示用户名为 zabbix 的用户，并且这个用户只能从 localhost 连接到 MySQL 服务器。localhost 通常指的是 MySQL 服务器所在的机器。

权限范围：这个权限仅限于zabbix数据库，不影响其他数据库。
```

<font style="color:rgb(51, 51, 51);">第三步：临时降低安全性</font>

```shell
#告诉 MySQL 信任你，即使你正在创建或修改一个非确定性函数，你也知道如何确保主服务器和从服务器之间的数据一致性。为了防止后面导入初始数据库时会出现报错，所以临时降低安全性。
mysql> set global log_bin_trust_function_creators = 1;

#退出
mysql> quit;
```

<code><font style="color:rgb(51, 51, 51);background-color:rgb(243, 244, 244);">set global log_bin_trust_function_creators = 1;</font></code><font style="color:rgb(51, 51, 51);">作用：我信任开发人员写的函数，请允许它们执行。</font>

<font style="color:rgb(51, 51, 51);">因为 Zabbix 安装时会在数据库里创建一些函数，而这些函数里用了像</font><code><font style="color:rgb(51, 51, 51);background-color:rgb(243, 244, 244);">NOW()</font></code><font style="color:rgb(51, 51, 51);">这样的语句，MySQL 默认是不让普通人创建这种函数的，怕有风险。</font>

<font style="color:rgb(51, 51, 51);">如果不设置，则结果就会导致 Zabbix 安装报错，从而安装失败。</font>

<font style="color:rgb(51, 51, 51);">设置它，就能解决这个问题，让安装顺利进行。</font>

<font style="color:rgb(51, 51, 51);">第四步：导入数据库</font>

```shell

Enter password:输入之前设置zabbix访问数据库的密码
```

<font style="color:rgb(51, 51, 51);">官方提供的命令行，大致意思是读取</font><code><font style="color:rgb(51, 51, 51);background-color:rgb(243, 244, 244);">/usr/share/zabbix-sql-scripts/mysql/server.sql.gz</font></code><font style="color:rgb(51, 51, 51);">文件（Zabbix 的 MySQL 初始化脚本）的内容，并将其直接导入到名为</font><code><font style="color:rgb(51, 51, 51);background-color:rgb(243, 244, 244);">zabbix</font></code><font style="color:rgb(51, 51, 51);">的 MySQ L数据库中，使用</font><code><font style="color:rgb(51, 51, 51);background-color:rgb(243, 244, 244);">zabbix</font></code><font style="color:rgb(51, 51, 51);">用户名和提示输入的密码进行身份验证，并确保在数据传输过程中使用</font><code><font style="color:rgb(51, 51, 51);background-color:rgb(243, 244, 244);">utf8mb4</font></code><font style="color:rgb(51, 51, 51);">字符集。</font>

## <font style="color:rgb(51, 51, 51);">创建自定义键值监控项</font>

<font style="color:rgb(51, 51, 51);">以监控登录用户数为例，自带键值中有</font><code><font style="color:rgb(51, 51, 51);background-color:rgb(243, 244, 244);">system.users.num</font></code><font style="color:rgb(51, 51, 51);">这个键值，但我们不使用它，使用自定义的键值来实现。</font>

### <font style="color:rgb(51, 51, 51);">在被监控端agent1上操作</font>

<font style="color:rgb(51, 51, 51);">首先在 agent1 多打开几个终端，模拟多个登录用户，然后使用</font><code><font style="color:rgb(51, 51, 51);background-color:rgb(243, 244, 244);">who | wc -l</font></code><font style="color:rgb(51, 51, 51);">查询</font>

```shell
[root@agent1 ~]# who | wc -l
2						
我这里确认登录用户数为2
```

<font style="color:rgb(51, 51, 51);">然后在 agent1 上，定义 UserParameter</font>

```shell
[root@agent1 ~]# vim /etc/zabbix/zabbix_agentd.conf

350行 UserParameter=loginusers,who | wc -l

说明: loginusers是我自定义的一个键值名称（会在创建监控项时用到), 后面的who | wc -l就是要被监控的命令
如果有多个自定义的键，那就写多个，比如:
UserParameter=loginusers,who | wc -l
UserParameter=meminfo,free -h
```

# <font style="color:rgb(51, 51, 51);">九、触发器</font>

<font style="color:rgb(51, 51, 51);">虽然我们可以通过图形查看到监控的数据，但我们不可能一直盯着图形的变化。</font>

<font style="color:rgb(51, 51, 51);">所以需要定义监控项到达一个临界值(阈值)或者满足一个条件，就会发生状态变化的通知。</font>

<font style="color:rgb(51, 51, 51);">定义</font>**<font style="color:rgb(51, 51, 51);">触发器(trigger)</font>**<font style="color:rgb(51, 51, 51);">就是定义这个临界值(阈值)或条件。</font>

<font style="color:rgb(51, 51, 51);">监控项有无限种可能，触发器也一样有无限种可能。如: </font>

* <font style="color:rgb(51, 51, 51);">cpu 负载值大于某个值则通知</font>
* <font style="color:rgb(51, 51, 51);">登录用户数大于某个值则通知</font>
* <font style="color:rgb(51, 51, 51);">内存空闲率小于某个值则通知</font>
* <font style="color:rgb(51, 51, 51);">磁盘使用率大于某个值则通知</font>
* <font style="color:rgb(51, 51, 51);">主机名被修改则通知</font>

<font style="color:rgb(51, 51, 51);">等等，主要还是看需求。</font>

# <font style="color:rgb(51, 51, 51);">十、报警</font>

<font style="color:rgb(51, 51, 51);">触发器的通知信息显示在 web 管理界面，运维工程师仍然没办法 24 小时盯着它。所以我们希望它能自动地通知工程师们，这就是报警。</font>

<font style="color:rgb(51, 51, 51);">zabbix 的报警媒介支持 email、jabber、sms(短信)、微信、电话语音等，新版本中也支持企微、钉钉等告警形式。</font>

## <font style="color:rgb(51, 51, 51);">报警过程原理</font>

![1765801885142-4f9483ca-c59d-40ca-95e1-fb0a3cd791bc.png](./img/9OvQbRGEUxqd9y86/1765801885142-4f9483ca-c59d-40ca-95e1-fb0a3cd791bc-361291.png)

## <font style="color:rgb(51, 51, 51);">报警平台申请与配置</font>

<font style="color:rgb(51, 51, 51);">自己配置报警过程比较复杂，需要配置触发器动作，用户与其报警媒介，最麻烦的是写程序对接邮件、微信、短信、电话等接口。</font>

* <font style="color:rgb(51, 51, 51);">邮件容易被拒，当做垃圾邮件</font>
* <font style="color:rgb(51, 51, 51);">微信需要企业微信号并开发程序对接</font>
* <font style="color:rgb(51, 51, 51);">短信一般都需要付费买运营商相关服务</font>
* <font style="color:rgb(51, 51, 51);">电话语音需要更专业的开发</font>

<font style="color:rgb(51, 51, 51);">以上要求对于没有开发能力和开发支持的运维工程师来说，难度较大。</font>

<font style="color:rgb(51, 51, 51);">所以我们这里选择专业的报警平台就可以帮助实现一体化报警方案。</font>

<font style="color:rgb(51, 51, 51);">如：onealeart，参考：</font>[<font style="color:rgb(65, 131, 196);">http://www.onealert.com/</font>](http://www.onealert.com/)

# <font style="color:rgb(51, 51, 51);">十一、自动化批量监控</font>

目前我们使用 Zabbix 监控主机的时候，需要做的事情：

1. 在被监控机器上安装 zabbix-agent 并做配置
2. 在 Zabbix Server 的 web 界面中添加要监控的主机信息
3. 在 web 界面中配置被监控的主机的模板、监控项等等

<font style="color:rgb(51, 51, 51);">我们要监控的服务器数量很大的情况下，如何批量操作:</font>

* <font style="color:rgb(51, 51, 51);">系统使用 cobbler 批量安装</font>
* <font style="color:rgb(51, 51, 51);">zabbix-agent 安装与配置可以使用 cobbler 的 postscript 脚本实现，或者使用 ansible 来实现</font>
* <font style="color:rgb(51, 51, 51);">监控主机元素: 监控项，图形，触发器等，统一使用模板</font>

<font style="color:rgb(51, 51, 51);">因为创建监控主机和添加模板都需要 web 界面操作，如何自动批量做?</font>

<font style="color:rgb(51, 51, 51);">答案: 自动发现或自动注册。</font>

# <font style="color:rgb(51, 51, 51);">十二、zabbix 代理</font>

## <font style="color:rgb(51, 51, 51);">zabbix proxy 应用场景</font>

<font style="color:rgb(51, 51, 51);">参考: </font>[<font style="color:rgb(65, 131, 196);">https://www.zabbix.com/documentation/current/manual/distributed\_monitoring/proxies</font>](https://www.zabbix.com/documentation/current/manual/distributed_monitoring/proxies)

Zabbix Proxy 使用场景：

* 监控远程区域设备
* 监控本地网络不稳定区域
* **当 Zabbix 监控上千设备时，使用它来减轻 Server 的压力**
* 简化分布式监控的维护

![1765876617552-5145e125-04a3-45a7-8d56-e18dd3098976.png](./img/9OvQbRGEUxqd9y86/1765876617552-5145e125-04a3-45a7-8d56-e18dd3098976-950246.png)

***

**<font style="color:rgb(51, 51, 51);">应用场景1: 跨内外网监控</font>**

<font style="color:rgb(51, 51, 51);">当 zabbix server 与被监控机器不在同一个机房时，跨公网监控会很麻烦，也会带来安全隐患</font>

* <font style="color:rgb(51, 51, 51);">比如有防火墙的情况，需要防火墙开放的端口增多</font>
* <font style="color:rgb(51, 51, 51);">像 mysql 数据库这类应用是不适合直接被公网连接的</font>

**<font style="color:rgb(51, 51, 51);">应用场景2: 分布式监控</font>**

<font style="color:rgb(51, 51, 51);">当监控机主机特别多，甚至分散在不同的地域机房。这个时候 zabbix server 压力很大，所以可以通过增加 zabbix proxy 来代理收集每个机房里的主机信息，再统一给 zabbix server。</font>

# <font style="color:rgb(51, 51, 51);">十三、主动监控与被动监控</font>

**<font style="color:rgb(51, 51, 51);">一共有 4 种模式:</font>**

1. <font style="color:rgb(51, 51, 51);">agent 主动</font>
2. <font style="color:rgb(51, 51, 51);">agent 被动(默认)</font>
3. <font style="color:rgb(51, 51, 51);">proxy 主动(默认)</font>
4. <font style="color:rgb(51, 51, 51);">proxy 被动</font>

## <font style="color:rgb(51, 51, 51);">agent 被动</font>

<font style="color:rgb(51, 51, 51);">相对于 agent 来说，也就是表示是 server 或 proxy 去找 agent 拿数据（agent 只要等待就可以了）。</font>

```shell
# grep -n ^Server= /etc/zabbix/zabbix_agentd.conf
Server=192.168.126.174						 agent被动模式, IP为server或proxy的IP
```

## <font style="color:rgb(51, 51, 51);">agent 主动</font>

<font style="color:rgb(51, 51, 51);">相对于 agent 来说，也就是表示是 agent 主动把数据传给 server 或 proxy。</font>

```shell
# grep -n ^ServerActive= /etc/zabbix/zabbix_agentd.conf
ServerActive=192.168.126.174			     agent主动模式, IP为server或proxy的IP
```

**<font style="color:rgb(51, 51, 51);">优点:</font>**<font style="color:rgb(51, 51, 51);"> 当 agent 太多的情况下，server 或 proxy 去找这么多 agent 搜集数据，压力负载过大。用 agent 主动模式就可以缓解 server 或 proxy 的压力。</font>

<font style="color:rgb(51, 51, 51);">但用主动模式的问题是：监控项也要转为主动式。</font>

<font style="color:rgb(51, 51, 51);">数据采集 => 模板 => 监控项</font>

# <font style="color:rgb(51, 51, 51);">十四、web 监测(拓展补充)</font>

<font style="color:rgb(51, 51, 51);">web 监测也是模板的一部分，当然也可以针对某一台主机（数据采集-主机）</font>

<font style="color:rgb(51, 51, 51);">主机里面进行配置：只有这台机器，配置了这个功能</font>

<font style="color:rgb(51, 51, 51);">模板里面进行配置：相当于所有使用了这个模板的机器都配置了这个功能</font>

**<font style="color:rgb(51, 51, 51);">web 监测</font>**<font style="color:rgb(51, 51, 51);">：类似一个 (可以包含多个小监控项)，主要针对 web 服务器做监控场景（httpd、nginx）。</font>

```shell
[root@agent1 ~]# yum install httpd -y

[root@agent1 ~]# systemctl start httpd
[root@agent1 ~]# systemctl enable httpd
```
