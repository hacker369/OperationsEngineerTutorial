## Node.js

### 介绍

Node.js 是一种基于 JavaScript 的运行环境，简单来说，它允许你在服务器端(而不是仅仅在浏览器中)运行 JavaScript 代码。你可以把它想象成一个让你用 JavaScript 做后端开发的工具。

作用:

* 构建服务器：通过 Node.js，开发者可以使用 JavaScript 来创建网站的后端(服务器部分)。它能帮助处理网站的请求、发送数据、连接数据库等任务。
* 快速高效：Node.js 使用非阻塞、事件驱动的架构，意味若它能同时处理大量的请求，效率很高，特别适合需要高并发的应用。
* 跨平台：它可以在 Windows、Mac 和 Linux 系统上运行。

# 三、项目部署（生产环境）

## 部署架构说明

目前我们已经将养老项目的前端、后端、小程序端等都部署的运行起来了（在Windows中）。接下来我们需要将他们部署在Linux中跑起来！

![1761609217916-1b9d69a9-a490-4c93-a61d-3a167f3cb93d.png](./img/8Q29NsTomVLQoBsD/1761609217916-1b9d69a9-a490-4c93-a61d-3a167f3cb93d-060661.png)

说明：

* 小程序服务不需要我们部署在Linux中，它需要将代码上传至微信公众平台，进行发布上线
* 后台服务我们需要将其打包，然后部署在Linux中。
  * 打包方式有两种，war包和jar包。如果是war包，就需要我们自己配置一个Tomcat服务器运行项目；如果是jar包，那么项目中已经内嵌了Tomcat服务器，不需要我们自己配置外部的Tomcat服务器了
  * 后台服务在Linux运行时，需要的环境有：JDK、MySQL、redis、阿里云OSS、百度千帆大模型；我们就差JDK没有配置好了（Maven是属于开发时用的环境，部署上线不需要使用）
* 前端服务，目前我们在Windows中通过node跑起来的，node属于开发环境用的东西；要将前端项目部署在Linux中运行，我们需要有Nginx服务器，将前端项目放入Nginx服务器中运行；然后Nginx需要配置反向代理，去访问后台服务！
* 小程序端给用户用，前端服务给工作人员用，后台服务是处理小程序端和前端服务的功能代码！

**总结1：需要在Linux中安装哪些内容？**

1. Nginx：运行前端项目
2. MySQL：后端项目要操作数据库
3. redis：后端项目要操作缓存，缓存里面放的是经常要查询的一些数据，这样的话查起来快
4. JDK：因为后端项目使用Java开发的
5. Tomcat/jar：就是用来运行Java项目的

**总结2：服务器节点数量：3台服务器**

第一台：专门用于存储数据的服务器：MySQL、redis

第二台：专门用户接收用户请求的服务器：Nginx

第三台：专门用于处理后台请求的服务器：Tomcat（也可以不装，如果是jar包的话内嵌了），JDK

多台服务器基础配置：

1. 三个服务器的ssh（免密）
2. 时间同步：以Nginx服务器也作为时间同步服务器，Nginx和外界进行时间同步，其他两台服务器以Nginx作为时间同步
3. 配置域名：测试环境下，仅在内部测试使用，所以需要配置内网DNS。上线后，需要购买真正的域名
4. 日志管理：将Nginx日志和Tomcat日志统一进行管理

## 环境准备

第一步：确保三台服务器的网卡配置要求

```shell
ip:
  节点一：192.168.126.171
  节点二：192.168.126.172
  节点三：192.168.126.173

子网掩码：255.255.255.0

DNS：8.8.8.8

通过ping命令测试网络是否畅通（每台机器都ping一下百度）
```

第二步：确保三台服务器的主机名（FQDN）

```shell
# hostnamectl set-hostname xxx.lhp.cn

节点一：node1.lhp.cn
节点二：node2.lhp.cn
节点三：node3.lhp.cn


三个节点的hosts文件统一添加：
127.0.0.1   localhost localhost.localdomain localhost4 localhost4.localdomain4
::1         localhost localhost.localdomain localhost6 localhost6.localdomain6
192.168.126.191 node1 node1.lhp.cn
192.168.126.192 node2 node2.lhp.cn
192.168.126.193 node3 node3.lhp.cn
```

第三步：确保三个节点SSH互通（也就是三个节点**互相做了免密**）

```shell
节点1的操作：
# ssh-keygen

# ssh-copy-id node1

# ssh-copy-id node2

# ssh-copy-id node3

# ssh node1
# exit

# ssh node2
# exit

# ssh node3
# exit
```

第四步：关闭SELinux深层防火墙

```shell
# setenforce 0
# vim /etc/selinux/config
22行 SELINUX=disabled
```

```shell
外部数据源：阿里云与腾讯云
```

`vim mysql.sh`

```properties
#!/bin/bash
#1.安装依赖软件
echo "正在安装依赖软件..."
yum -y install libaio &> /dev/null
if [ $? -ne 0 ];then
    echo "libaio安装失败"
    exit 1
fi
#2.判断是否有压缩包，如果有，则执行解压缩操作
echo "正在判断是否有压缩包，如果有进行解压缩操作..."
if [ -f mysql-8.0.40-linux-glibc2.17-x86_64.tar.xz ]; then
    tar -xf mysql-8.0.40-linux-glibc2.17-x86_64.tar.xz
    ls -l mysql-8.0.40-linux-glibc2.17-x86_64
fi
#3.判断系统中是否安装过mariadb软件，如果有对其进行卸载操作
echo "正在判断系统中是否安装过mariadb软件，如果有对其进行卸载操作..."
rpm -qa | grep mariadb | xargs -r dnf remove -y
[ -f /etc/my.cnf ] && rm -rf /etc/my.cnf
#4.创建mysql系统账号
id mysql &> /dev/null
[ $? -ne 0 ] && useradd -r -s /sbin/nologin mysql
#5.创建/export/server目录，然后移动mysql压缩包解压后的文件到/export/server目录下
rm -rf /export/server
mkdir -p /export/server
mv mysql-8.0.40-linux-glibc2.17-x86_64 /export/server/mysql
#6.进入mysql目录，对其进行初始化操作
echo "正在进入mysql目录，对其进行初始化操作..."
cd /export/server/mysql
bin/mysqld --initialize --user=mysql --basedir=/export/server/mysql --datadir=/export/server/mysql/data 2>&1 | tee /tmp/mysqld.log | grep password | awk '{print $NF}' > /tmp/mysql_temp_password.txt
#7.设置ssl加密传输连接
bin/mysql_ssl_rsa_setup --datadir=/export/server/mysql/data &> /dev/null
#8.设置my.cnf与mysqld.service文件
echo "正在设置my.cnf与mysqld.service文件..."
cat > /etc/my.cnf <<EOF
[mysqld]
port=3306
basedir=/export/server/mysql
datadir=/export/server/mysql/data
socket=/tmp/mysql.sock
character_set_server=utf8
collation-server=utf8_unicode_ci
EOF

cat > /etc/systemd/system/mysqld.service <<EOF
[Unit]
Description=MySQL Server
After=network.target

[Service]
User=mysql
Group=mysql
Type=forking

# MySQL 执行命令及路径
ExecStart=/export/server/mysql/bin/mysqld --daemonize --pid-file=/export/server/mysql/data/mysqld.pid
ExecStop=/export/server/mysql/bin/mysqladmin --defaults-file=/export/server/mysql/my.cnf shutdown

# Ensure MySQL has sufficient time to start up
TimeoutSec=600

# PID 文件路径
PIDFile=/export/server/mysql/data/mysqld.pid

# Enable these options to auto-restart the service if it crashes
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF
#9.刷新后台服务，然后启动mysqld
echo "正在刷新后台服务，然后启动mysqld..."
systemctl daemon-reload
systemctl start mysqld
systemctl enable mysqld
#10.重置mysql管理员密码为123456
echo "正在重置mysql管理员密码..."
cd /export/server/mysql
temp_password=`cat /tmp/mysql_temp_password.txt`
bin/mysqladmin -uroot password '123456' -p$temp_password
#11.把mysql的bin目录添加到环境变量中
echo 'export PATH=$PATH:/export/server/mysql/bin' >> /etc/profile
source /etc/profile
#12.解决mysql客户端首次无法登录问题
[ ! -f /lib64/libncurses.so.5 ] && ln -s /lib64/libncurses.so.6 /lib64/libncurses.so.5
[ ! -f /lib64/libtinfo.so.5 ] && ln -s /lib64/libtinfo.so.6 /lib64/libtinfo.so.5
#13.弹出提示，MySQL安装成功
echo "MySQL安装成功，软件安装路径：/export/server/mysql，数据库初始密码：123456！"
```
