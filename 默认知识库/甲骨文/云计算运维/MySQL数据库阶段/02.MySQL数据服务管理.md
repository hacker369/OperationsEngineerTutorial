常见连接异常说明：

排查1：排查 IP 地址是否可以 ping 通

Windows，cmd：

```shell
ping 192.168.126.101
```

排查2：查看 MySQL 端口是否被占用

```shell
ss -nltp

查看mysqld进程是否占用了3306端口
```

特殊：MySQL 软件可以正常使用，但是 ss 查看没有端口，这就是 MySQL 启动异常（有进程，没有端口），解决思路

```shell
pkill mysqld
systemctl start mysqld
systemctl enable mysqld
```

排查3：查看服务器防火墙是否开启，如果开启，是否放行 3306、33060 端口

```shell
systemctl stop firewalld
systemctl disable firewalld
```

排查4：除了以上配置以外，还需要有一个远程账号 root@%

```shell
mysql> select user,host from mysql.user;
```

# 二、SQL 语句（重点）

## SQL 语句概述

<font style="color:rgb(216,57,49);">结构化查询语言</font>(Structured Query Language)简称 <font style="color:rgb(216,57,49);">SQL</font>，是<font style="color:rgb(216,57,49);">关系型数据库管理系统</font>都需要遵循的<font style="color:rgb(216,57,49);">规范</font>，是数据库认识的语句。不同的数据库生产厂商都<font style="color:rgb(216,57,49);">支持 SQL 语句</font>，但都有<font style="color:rgb(216,57,49);">特有内容</font>。

**举例：**

普通话：各数据库厂商都遵循的 ISO 标准。

方言：数据库特有的关键字。

![1769919702207-7c8588ba-95a5-4ad9-9990-b25d8b23c932.png](./img/4ZrKvESLqUG3Y-Dc/1769919702207-7c8588ba-95a5-4ad9-9990-b25d8b23c932-998525.png)

## SQL 语句分类（记住）

### DDL

<font style="color:rgb(216,57,49);">数据定义语言</font>：简称 DDL（Data Definition Language）

用来定义数据库对象：<font style="color:rgb(216,57,49);">数据库，表，列等</font>。

关键字：<font style="color:rgb(216,57,49);">create，alter，drop </font>等

> 搭建数据存储的框架，如管理数据库、管理数据表、管理字段

### DML

<font style="color:rgb(216,57,49);">数据操作语言</font>：简称 DML（Data Manipulation Language）

用来对数据库中表的记录进行更新（<font style="color:rgb(216,57,49);">增加、修改、删除</font>）。

关键字：<font style="color:rgb(216,57,49);">insert，delete，update </font>等

### DQL

<font style="color:rgb(216,57,49);">数据查询语言</font>：简称 DQL（Data Query Language）

用来查询数据库中表的记录（<font style="color:rgb(216,57,49);">查询</font>）。

关键字：select，from，where 等

### DCL

<font style="color:rgb(216,57,49);">数据控制语言</font>：简称 DCL（Data Control Language）

用来定义数据库的<font style="color:rgb(216,57,49);">访问权限和安全级别及创建用户（账号管理、权限管理）</font>。

## SQL 基本语法

① SQL 语句可以单行或多行书写，但是最终需要以<font style="color:rgb(216,57,49);">分号结尾</font>。

```sql
select * from students;
```

② 可使用<font style="color:rgb(216,57,49);">空格和缩进</font>来<font style="color:rgb(216,57,49);">增强</font>语句的<font style="color:rgb(216,57,49);">可读性</font>

```sql
select
  *
from students;
```

③ MySQL 数据库的 SQL 语句默认<font style="color:rgb(216,57,49);">不区分大小写</font>

```sql
SELECT * FROM user;
等价于
select * from user; 
```

④ 可以使用<font style="color:rgb(216,57,49);">单行与多行注释</font>

```sql
#  单行注释
-- 单行注释，注意：--后面有一个空格

/*
  ...
  多行注释
  ...
*/
```

## 小结

SQL：（<font style="color:rgb(216,57,49);">结构化查询语言</font>），是关系型数据库管理系统都需要遵循的规范，不同的数据库生产厂商都支持 SQL 语句，但都有特有内容。

SQL 语句分为：数据定义语言（DDL），数据操作语言（DML），数据查询语言（DQL），数据控制语言（DCL）。

SQL 通用语法：① 分号结尾 ② 空格或者缩进 ③ 不区分大小写 ④ 注释：单行注释 与 多行注释

# 三、DDL 数据库操作

## MySQL 的组成结构（理解）

![1769996224427-8babaf71-ad55-40c6-907a-6d3705cfb882.png](./img/4ZrKvESLqUG3Y-Dc/1769996224427-8babaf71-ad55-40c6-907a-6d3705cfb882-652276.png)

> 注：我们平常说的 MySQL，其实主要指的是 MySQL 数据库管理软件。

<font style="color:rgb(216,57,49);">一个 MySQL DBMS 可以同时存放多个数据库</font>，理论上<font style="color:rgb(216,57,49);">一个项目就对应一个数据库</font>。如博客项目 blog 数据库、商城项目 shop 数据库、微信项目 wechat 数据库。

<font style="color:rgb(216,57,49);">一个数据库中还可以同时包含多个数据表，而数据表才是真正用于存放数据的位置</font>。（类似我们 Office 软件中的 Excel 表格），理论上<font style="color:rgb(216,57,49);">一个功能就对应一个数据表</font>。如博客系统中的用户管理功能，就需要一个 user 数据表、博客中的文章就需要一个 article 数据表、博客中的评论就需要一个 message 数据表。

<font style="color:rgb(216,57,49);">一个数据表又可以拆分为多个字段</font>，每个字段就是一个<font style="color:rgb(216,57,49);">属性</font>。一个用户表，需要保存用户账号、用户的密码等信息。

<font style="color:rgb(216,57,49);">一个数据表除了字段以外，还有很多行，每一行都是一条完整的数据</font>（<font style="color:rgb(216,57,49);">记录</font>）。

## 数据库的基本操作

### 创建数据库

普及英语小课堂：

创建 => <font style="color:rgb(216,57,49);">create</font>

数据库 => <font style="color:rgb(216,57,49);">database</font>

创建 + 数据库 = create database 数据库名称(字母+数字+下划线组成，以字母开头，不能出现中文以及特殊字符)

基本语法：

```sql
mysql> create database 数据库名称 [设置编码格式];
```

> 特别注意：在 MySQL 中，当一条 SQL 语句编写完毕后，一定要使用分号;进行结尾，否则系统认为这条语句还没有结束。

案例：创建数据库的相关案例

```sql
创建db_lhp库
create database db_lhp;

创建db1库并指定默认字符集
create database db1 default charset gbk;

如果存在不报错(if not exists)
create database if not exists db_lhp default character set utf8;
说明：不能创建相同名字的数据库！
```

> 扩展：编码格式，常见的 gbk（中国的编码格式）与 utf8（国际通用编码格式）

latin1：这是早期的字符编码格式，支持 256 个字符，适用于基本的西方语言字符集。

gb2312/gbk：由于 latin1 的字符集无法表示汉字，中国开发了自己的编码格式 gb2312，后来升级为 gbk，以支持更多汉字和扩展字符集。

big5：中国台湾也开发了一套用于表示繁体中文的字符编码格式，称为 big5，适用于繁体中文字符集。

utf8：为了支持多语言环境，国际标准化组织开发了 utf8 编码，能够表示几乎所有已知语言的字符。由于早期版本的 utf8 不完全支持某些特殊符号和表情符号，MySQL5.6 版本以后推出了 utf8mb4，用于支持更多字符（如 Emoji 表情），每个字符最多使用 4 个字节。

## 数据表的基本操作

### 数据表的创建

英语小课堂：

创建 => <font style="color:rgb(216,57,49);">create</font>

数据表 => <font style="color:rgb(216,57,49);">table</font>

创建 + 数据表 = create table 数据表名称

基本语法：

```sql
mysql> create table 数据表名称(
        字段1 字段类型 [字段约束],
        字段2 字段类型 [字段约束],
        ...
); 

常用字段类型
整数：int（-21亿 ~ 21亿）
浮点：decimal(M,N)，M代表整数长度 + 小数总长度，N代表小数点后保留几位小数，decimal(5,2) => 999.99
字符串类型：固定长度用char(M)，变化长度用varchar(M) => M代表最大字符数
枚举类型：enum('男','女')，插入值要么为男，要么为女
```

案例：创建一个 admin 管理员表，拥有 3 个字段（编号、用户名称、用户密码）

```sql
mysql> create database db_lhp;
mysql> use db_lhp;
```

> use 在 MySQL 中的含义代表选择，use 数据库名称相当于选择指定的数据库。而且 use 比较特殊，其选择结束后，其尾部可以不加分号；但是强烈建议所有的 SQL 语句都要加分号，养成一个好习惯。

```sql
mysql> create table tb_admin(
    id int,
    username varchar(20),
    password varchar(20)
) default charset=utf8;
```

案例：创建一个 article 文章表，拥有4个字段（编号、标题、作者、内容）

```sql
mysql> use db_lhp;
mysql> create table tb_article(
    id int,
    title varchar(50),
    author varchar(20),
    content text
) default charset=utf8;
```

> text：文本类型，一般情况下，用 varchar 存储不了的字符串信息，都建议使用 text 文本进行处理。
>
> varchar 存储的最大长度，理论值 65535 个字节。但是实际上，有几个字节是用于存放内容的长度的，所以真正可以使用的不足65535 个字节，另外 varchar 类型存储的字符长度还和编码格式有关。1个 GBK 格式的占用2个字节长度，1个 UTF8 格式的字符占用3个字节长度。GBK = 65532~65533/2，UTF8 = 65532~65533/3

案例：创建一个 student 表，字段：编号、姓名、性别、年龄、体重

```properties
create table student(
  id int,
  name varchar(10),
  sex enum("男", "女"),
  age int,
  weight decimal(5,2)
);
```

## 字段约束详解（重点）

字段约束：在<font style="color:rgb(216,57,49);">数据类型</font>的基础上，给字段添加<font style="color:rgb(216,57,49);">约束行为</font>（如不能为空等等）

基本语法：

```sql
create table 数据表名称(
     字段名称  字段类型  [可选，字段约束]
) default charset=utf8mb4;
```

在 MySQL 中，字段约束一共有五种约束：<font style="color:rgb(216,57,49);">① 主键约束 ② 唯一约束 ③ 非空约束 ④ 默认值约束 ⑤ 外键约束</font>

### 主键约束

强调：<font style="color:rgb(216,57,49);">非空、唯一、一个表中有且仅有1个主键</font>，在 MySQL 中，往往 <font style="color:rgb(216,57,49);">id 字段</font>都是主键（<font style="color:rgb(216,57,49);">primary key</font>）

article 文章表（id 主键，title 标题，author 作者，addtime 发布时间）

```sql
use db_lhp;
create table article(
    id int primary key,
    title varchar(80),
    author varchar(20),
    addtime date
) default charset=utf8mb4;

或

create table article(
    id int,
    title varchar(80),
    author varchar(20),
    addtime date,
    primary key(id)
) default charset=utf8mb4;
```

案例：

```sql
use db_lhp;
create table article(
    id int primary key,
    title varchar(80),
    author varchar(20),
    addtime date
) default charset=utf8mb4;

-- 查看数据表
desc article;

-- 向表中插入数据
insert into article values (1, 'MySQL数据库运维', '张三', '2025-02-10');

-- 主键强制为null（错误演示）
insert into article values (null, 'Python运维开发', '李四', '2025-02-10');

-- 主键强制重复（错误演示）
insert into article values (1, 'Python运维开发', '李四', '2025-02-10');
```

### 唯一约束

强调：<font style="color:rgb(216,57,49);">唯一，可以为空，一个表中可以同时拥有多个唯一约束</font>

unique

主键约束 与 唯一约束区别？

和主键约束相同点：<font style="color:rgb(216,57,49);">都是唯一的</font>

和主键约束不同点：<font style="color:rgb(216,57,49);">主键非空、有且仅有1个</font>；<font style="color:rgb(216,57,49);">唯一可以为空，在一个表中也可以同时拥有多个</font>

案例：创建一个用户表，id 主键，name，age，code（身份证号码，要求唯一）

```sql
create table users(
    id int primary key,
    name varchar(20),
    age tinyint unsigned,
    code char(18) unique
) default charset=utf8;

-- 插入数据
insert into users values (1, '张三', 18, '123456789012345678');

-- 插入重复数据（错误演示）
insert into users values (2, '李四', 18, '123456789012345678');

-- 插入数据，但是唯一列为null
insert into users values (2, '王五', 18, null);
```

### 非空约束

强调：<font style="color:rgb(216,57,49);">非空</font>，这个列不允许出现 null 空值

<font style="color:rgb(216,57,49);">not null</font>

案例：创建一个商品表 goods，有 id 主键，name 商品名称，price 商品价格，name 和 price 都必须为非空！

```sql
create table goods(
    id int primary key,
    name varchar(80) not null,
    price decimal(11,2) not null
) default charset=utf8mb4;

-- 插入数据
insert into goods values (1, 'Python从入门到放弃', 99.99);

-- 插入数据（错误演示）
insert into goods values (2, 'Java从入门到放弃', null);
```

### 默认值约束

强调：<font style="color:rgb(216,57,49);">默认值</font>，插入数据时，如果没有指定对应的值，则其自动使用默认值做为最终结果

<font style="color:rgb(216,57,49);">default 默认值</font>

案例：创建一个用户表 users，id 主键、name 用户名、age 年龄、gender 性别，要求 gender 默认值为 male

```sql
drop table users;

create table users(
  id int primary key,
  name varchar(20),
  age tinyint unsigned,
  gender enum('male', 'female') default 'male'
) default charset=utf8mb4;

desc users;

-- 插入数据
insert into users values (1, '张三', 18, 'male');

-- 插入数据（触发默认值）
insert into users(id, name, age) values (2, '李四', 18);
insert into users values (3, '王五', 18, default);

-- 插入数据（不使用默认值）
insert into users values (4, '赵六', 18, 'female');
```

### 数据的增加操作

基本语法：

```sql
mysql> insert into 数据表名称([字段1,字段2,字段3...]) values (字段1的值,字段2的值,字段3的值...);
```

> 特别注意：<font style="color:rgb(216,57,49);">在 SQL 语句中，除了数字，其他类型的值，都需要使用引号引起来，否则插入时会报错。</font>

<font style="color:#000000;">第一步：准备一个数据表</font>

```sql
mysql> use db_lhp;

      gender enum('男','女','保密'),
```

> unsigned 代表无符号型，只有0到正数。tinyint unsigned 无符号型，范围 0 ~ 255
>
> enum 枚举类型，多选一。只能从给定的值中选择一个

第二步：使用 insert 语句插入数据

```sql
mysql> insert into tb_user values (null,'刘备',34,'男','广州市天河区');

```

第三步：批量插入多条数据

```sql

```

### 数据的修改操作

基本语法：

```sql
mysql> update 数据表名称 set 字段1=更新后的值,字段2=更新后的值,... [where 更新条件];

where：更新条件，满足条件的才会被更新，如果不写where则默认修改所有记录！
```

> 特别说明：如果在更新数据时，不指定更新条件，则其会把这个数据表的所有记录全部更新一遍。

案例：修改 name='赵飞' 这条记录，将其性别更新为男，家庭住址更新为广东省深圳市

```sql
mysql> update tb_user set gender='男',address='广东省深圳市' where name='赵飞';
```

案例：今年是2025年，假设到了2026年，现在存储的学员年龄都差1岁，整体进行一次更新

```sql
mysql> update tb_user set age=age+1;
```

### 数据的删除操作

基本语法：

```sql
mysql> delete from 数据表名称 [where 删除条件];
```

案例：删除 tb\_user 表中，id=1 的用户信息

```sql
mysql> delete from tb_user where id=1;
```

delete from 与 truncate 清空数据表操作

```sql
mysql> delete from 数据表;
或
mysql> truncate 数据表;
```

面试题：<font style="color:rgb(216,57,49);">delete from 与 truncate 区别在哪里？</font>

* delete：删除数据记录
  * <font style="color:rgb(216,57,49);">数据操作语言（DML）</font>
  * <font style="color:rgb(216,57,49);">删除大量记录速度慢，只删除数据，主键自增序列不清零</font>，1-100 => 新插入 => 101
  * 可以带条件删除
* truncate：删除所有数据记录
  * <font style="color:rgb(216,57,49);">数据定义语言（DDL）</font>
  * <font style="color:rgb(216,57,49);">清理大量数据速度快，主键自增序列清零</font>, 1-100 => 新插入 => 1
  * 不能带条件删除

小结：

DML：数据操作语言，主要涉及增加（<font style="color:rgb(216,57,49);">INSERT</font>）、修改（<font style="color:rgb(216,57,49);">UPDATE</font>）、删除（<font style="color:rgb(216,57,49);">DELETE</font>）

```sql
五子句可以单独出现，也可以多个关键词一起出现。但是不管多少个关键字都必须严格按照五子句顺序进行书写，否则报错！！！
```

## 聚合查询

作用：之前我们做的查询都是横向查询，它们都是根据条件一行一行的进行判断，而使用聚合函数查询是纵向查询，它是对一列的值进行计算，然后返回一个单一的值；另外聚合函数会忽略空值。

聚合查询：<font style="color:rgb(216,57,49);">① 纵向查询（按列查询） ② 默认会忽略空值</font>

今天我们学习如下五个聚合函数：

| min() | 计算指定列的最小值，如果指定列是字符串类型，使用字符串排序运算； |
| avg() | 计算指定列的平均值，如果指定列类型不是数值类型，则计算结果为0 |

光有分组一般没有特别的意义，但是有一个特性：去重

### 分组查询介绍

分组查询就是将查询结果按照指定字段进行分组，字段中数据相等的分为一组。

**分组查询基本的语法格式如下：**

<font style="color:rgb(216,57,49);">GROUP BY 列名</font> \[<font style="color:rgb(216,57,49);">HAVING 条件表达式</font>]

**说明:**

* 列名: 是指按照指定字段的值进行分组。
* HAVING 条件表达式: 用来过滤分组后的数据。

### group by 的使用

group by 可用于单个字段分组，也可用于多个字段分组

```sql
-- 准备学生表student
create table student(
    id int auto_increment,
    name varchar(20),
    age tinyint unsigned,
    gender enum('男', '女'),
    score tinyint,
    primary key(id)
) default charset=utf8;

insert into student values (null, 'Tom', 23, '男', 97);
insert into student values (null, 'Jack', 24, '男', 88);
insert into student values (null, 'Rose', 26, '女', 99);
insert into student values (null, 'Eric', 27, '男', 59);
insert into student values (null, 'Jennifer', 22, '女', 76);

-- 根据gender字段来分组
select gender from student group by gender;

-- 根据name和gender字段进行分组
select name, gender from student group by name, gender;
```

① group by 可以实现去重操作

② group by 的作用是为了实现分组统计（group by + 聚合函数）

## having 的使用

having 作用和 where 类似都是<font style="color:rgb(216,57,49);">过滤数据</font>的，但是两者之间的<font style="color:rgb(216,57,49);">执行顺序不同</font>

<font style="color:rgb(216,57,49);">① where 子句（发生在分组之前）</font> ② group by 子句 <font style="color:rgb(216,57,49);">③ having 子句（发生在分组之后）</font>

第一种情况：如果只是简单的查询操作（<font style="color:rgb(216,57,49);">没有 group by 的情况</font>），<font style="color:rgb(216,57,49);">大部分时间 having 是可以直接替代 where 子句</font>

```sql
select * from product where price > 800;
```

以上语句等价于

```sql
select * from product having price > 800;
```

第二种情况：

```sql
-- 根据gender字段进行分组，统计分组条数大于2的
select gender,count(*) from students group by gender having count(*)>2;
```

案例演示：

```sql
#1 统计各个分类商品的个数
SELECT category_id ,COUNT(*) FROM product GROUP BY category_id ;

#2 统计各个分类商品的个数,且只显示个数大于1的信息
SELECT category_id ,COUNT(*) FROM product GROUP BY category_id HAVING COUNT(*) > 1;

#3 统计各个商品分类中，商品的价格大于500的商品的个数
select category_id, count(*) from product where price>500 group by category_id;
```

## limit 子句的使用

应用场景：① <font style="color:rgb(216,57,49);">限制查询 </font>② <font style="color:rgb(216,57,49);">分页查询</font>

限制查询：主要限制数据查询的数量（获取数据表中的前3条数据）

`select * from 数据表 limit 查询数量;`

`select * from 数据表 limit 偏移量(从哪查默认为 0，类似索引下标), 查询数量`

偏移量：索引下标，默认从0开始

0 第一条记录

1 第二条记录

2 第三条记录

```sql
-- 基本语法1：select * from 数据表 limit 查询数量;
-- 案例1：查询学生表中，成绩最高的学生信息（只查1个）
select * from student order by score desc limit 0,1;
select * from student order by score desc limit 1;
-- 缺点说明：如果一个学生表有多个同学成绩相同，可能会导致，只能查出1名同学。

-- 基本语法2：select * from 数据表 limit 偏移量(从哪查默认为0，类似索引下标)，查询数量;
-- 案例2：查询学生表中，成绩第2高（第2名）的学生信息
select * from student order by score desc limit 1,1;

-- 案例3: 找出学生中成绩是倒数后3名同学的信息
select * from student order by score asc limit 0,3;
```

分页查询：实际特别接近，因为只要有分页的地方，底层100%都是使用limit子句实现的

## 表与表之间的关系

在 SQL 语句中，数据表与数据表之间，如果存在关系，一般一共有3种情况：

① <font style="color:rgb(216,57,49);">1：1，一对一关系（高级）</font>

比如有A、B两张表，A表中的每一条数据，在B表中有一条唯一的数据与之对应。

用户表 user

| user\_id（用户编号） | 账号username | 密码password |
| --- | --- | --- |
| 001 | zhangsan | 888888 |
| 002 | lisi | 123456 |

用户详情表 user\_items

| user\_id（用户编号） | 真实姓名 | 年龄 | 联系方式 |
| --- | --- | --- | --- |
| 001 | 张三 | 16 | 10086 |
| 002 | 李四 | 18 | 10010 |

我们把用户表与用户详情表之间的关系就称之为一对一关系。

② <font style="color:rgb(216,57,49);">1：N，一对多关系</font>（<font style="color:rgb(216,57,49);">重点</font>）

比如有A、B两张表，A表中的<font style="color:rgb(216,57,49);">每一条数据</font>，在B表中<font style="color:rgb(216,57,49);">都有多条数据与之对应</font>，我们把这种关系就称之为<font style="color:rgb(216,57,49);">一对多</font>关系

| 2 | Redmi Note 9 | 3499.00 | 1 |

我们把产品分类表与产品表之间的关系就称之为一对多关系。

③ <font style="color:rgb(216,57,49);">M：N，多对多关系</font>（<font style="color:rgb(216,57,49);">高级</font>）

用户表

| 用户编号 | 登录账号 | 登录密码 |

<font style="color:rgb(216,57,49);">没有意义，但是它是所有连接的基础</font>。其功能就是将表1和表2中的每一条数据进行连接。

## 内连接（重点）

### 连接查询的介绍

连接查询可以实现多个表的查询，当查询的<font style="color:rgb(216,57,49);">字段数据</font>来自不同的表就可以使用连接查询来完成。

连接查询可以分为:

1. <font style="color:rgb(216,57,49);">内连接查询</font>
2. <font style="color:rgb(216,57,49);">左外连接查询</font>
3. <font style="color:rgb(216,57,49);">右外连接查询</font>
4. <font style="color:rgb(216,57,49);">自连接查询（自己关联自己）</font>

### 内连接查询

查询两个表中符合条件的共有记录

![1770106410236-ba2cada9-59a4-4715-b7e0-cc19e2cb8db4.png](./img/4ZrKvESLqUG3Y-Dc/1770106410236-ba2cada9-59a4-4715-b7e0-cc19e2cb8db4-933038.png)

**内连接查询语法格式: inner join ... on 关联条件**

```sql
select 字段 from 表1 inner join 表2 on 表1.字段1 = 表2.字段2
```

**说明:**

* inner join 就是内连接查询关键字
* on 就是连接查询条件

**例1：使用内连接查询学生表与班级表，查询每个学生对应的具体班级信息。**

```sql
-- 案例：求每个学生所属的班级信息（学生所有字段 + 班级名称）
select * from students inner join classes on students.cls_id = classes.cls_id;

-- 内连接基础上可以筛选想要的字段
select students.*, classes.cls_name from students inner join classes on students.cls_id = classes.cls_id;

-- 引入表别名机制，数据表名称 [as] 别名
select s.name, s.age, s.gender, c.cls_name from students s inner join classes c on s.cls_id = c.cls_id;

-- 通过换行+缩进提升可读性，inner join on还可以简写为join on
select
  s.name,
  s.age,
  s.gender,
  c.cls_name
from students as s
join classes as c
on s.cls_id = c.cls_id;
```

### 小结

* 内连接使用 inner join .. on .., on 表示两个表的连接查询条件
* 内连接根据连接<font style="color:rgb(216,57,49);">查询条件</font>取出两个表的 “<font style="color:rgb(216,57,49);">交集</font>”，也就是满足关联条件的结果。

## 左外连接

### 左连接查询

> 有一个主表概念，默认情况下，关联后会保留主表的所有记录。

以左表为主根据条件查询右表数据，如果根据条件查询右表数据不存在使用 null 值填充

商品分类表

| 编号 | 分类名称 |
| --- | --- |
| 1 | 手机 |
| 2 | 电脑 |

商品表

| 编号 | 商品名称 | 商品分类编号 |
| --- | --- | --- |
| 1 | Vivo手机 | 1 |
| 2 | Xiaomi手机 | 1 |

左外连接查询：select \* from <font style="color:rgb(216,57,49);">分类表（主表）</font> left join 商品表 on 分类表.编号 = 商品表.分类编号;

| 编号 | 分类名称 | 编号 | 商品名称 | 商品分类编号 |
| --- | --- | --- | --- | --- |
| 1 | 手机 | 1 | Vivo手机 | 1 |
| 1 | 手机 | 2 | Xiaomi手机 | 1 |
| 2 | 电脑 | null | null | null |

<font style="color:rgb(216,57,49);">左外连接默认会保留左表，然后与右边的表进行匹配；如果匹配到，则显示右表对应的数据；匹配不到，也要显示。只不过右表的所有字段使用 null 进行填充！</font>

![1770107175356-33ce1e09-34c4-42fe-98cb-06733610b699.png](./img/4ZrKvESLqUG3Y-Dc/1770107175356-33ce1e09-34c4-42fe-98cb-06733610b699-008986.png)

**左连接查询语法格式:**

```sql
select 字段 from 表1 left join 表2 on 表1.字段1 = 表2.字段2
```

**说明:**

* left join 就是左连接查询关键字
* on 就是连接查询条件
* 表1 是左表
* 表2 是右表

**例1：使用左连接查询学生表与班级表，获取每一学生对应的班级信息，要求没有对应的班级也要显示。**

```sql
-- 使用左连接查询学生表与班级表，获取每一学生对应的班级信息，要求没有对应的班级也要显示。
insert into students values (6, '曹操', 30, 'male', 98, 9);

select s.*,c.cls_name from students s left join classes c on s.cls_id = c.cls_id;
```

### 小结

* 左连接使用 left join .. on .., on 表示两个表的连接查询条件
* 左连接以左表为主根据条件查询右表数据，右表数据不存在使用 null 值填充。

## 右外连接

### 右连接查询

以<font style="color:rgb(216,57,49);">右表为主</font>根据条件查询左表数据，如果<font style="color:rgb(216,57,49);">根据条件查询左表数据</font>不存在则使用 <font style="color:rgb(216,57,49);">null 值</font>填充

![1770170585979-dc4e077f-b8db-4f96-b1a6-32d747b5d79a.png](./img/4ZrKvESLqUG3Y-Dc/1770170585979-dc4e077f-b8db-4f96-b1a6-32d747b5d79a-205113.png)

**右连接查询语法格式:**

```sql
select 字段 from 表1 right join 表2（主表） on 表1.字段1 = 表2.字段2
```

**说明:**

* right join 就是右连接查询关键字
* on 就是连接查询条件
* 表1 是左表
* 表2 是右表

**例1：使用右连接查询学生表与班级表:**

```sql
-- 查看班级中对应的学生信息，如果某个班级没有对应的学生也要显示。（既可以用左外连接也可以用右外连接）
select s.*,c.cls_name from students s right join classes c on s.cls_id = c.cls_id;
select s.*,c.cls_name from classes c right join students s on c.cls_id = s.cls_id;
```

### 小结

* 右连接使用 right join .. on .., on 表示两个表的连接查询条件
* 右连接以右表为主根据条件查询左表数据，左表数据不存在使用 null 值填充。

## 自连接查询(扩展)

自连接查询：<font style="color:rgb(216,57,49);">数据表自己连接自己</font>，本质只有1张数据表，表内部数据有层级关系。

前提：<font style="color:rgb(216,57,49);">连接操作时必须为数据表定义别名！</font>

左表和右表是同一个表，根据连接查询条件查询两个表中的数据。

两个实际的工作场景：求<font style="color:rgb(216,57,49);">省市区</font>信息，求<font style="color:rgb(216,57,49);">分类导航</font>信息

<font style="color:rgb(216,57,49);">① 省份表 + 城市表 + 区域表（理论，实际设计表的过程中，通常只有1张数据表）</font>

<font style="color:rgb(216,57,49);">② 大类 + 中类 + 小类（理论，实际设计表的过程中，通常只有1张数据表）</font>

***

地域：title

pid 全称 <font style="color:rgb(216,57,49);">parent id</font>（父级ID编号），如果pid值为null代表本身就是父级，如果pid是一个具体的数值，则代表其属于子级

**例1：查询省的名称为“广东省”的所有城市**

**创建 areas 表:**

```sql
use db_lhp;
create table areas(
    aid int not null AUTO_INCREMENT,
    atitle varchar(20),
    pid int,
    primary key(aid)
) default charset=utf8;
```

**执行 sql 文件给 areas 表导入数据:**

```sql
insert into areas values (null, '广东省', null),(null, '山西省', null),(null, '深圳市', 1), (null, '广州市', 1), (null, '太原市', 2), (null, '大同市', 2);
```

![1770172126597-95cecf51-0082-461b-969c-ae3b1e06b2d8.png](./img/4ZrKvESLqUG3Y-Dc/1770172126597-95cecf51-0082-461b-969c-ae3b1e06b2d8-171183.png)

**自连接查询的用法:**

```sql
-- 第一步：把两张表关联查询（内连接）
select * from areas p inner join areas c on p.aid = c.pid;

-- 第二步：在以上查询基础上，获取省份名称为广东省
select * from areas p inner join areas c on p.aid = c.pid where p.atitle = '广东省';

-- 第三步：只显示城市信息
select c.* from areas p inner join areas c on p.aid = c.pid where p.atitle = '广东省';
```

**说明:**

* 自连接查询必须对表起别名

### 小结

* 自连接查询就是把一张表模拟成左右两张表，然后进行连表查询。
* 自连接就是一种特殊的连接方式，连接的表还是本身这张表

# 八、子查询（了解）

## 子查询（嵌套查询）的介绍

在一个 select 语句中，<font style="color:rgb(216,57,49);">嵌入了另外一个 select 语句</font>，那么被嵌入的 select 语句称之为<font style="color:rgb(216,57,49);">子查询语句</font>，外部那个 select 语句则称为<font style="color:rgb(216,57,49);">主查询语句。</font>

<code>select * from (<font style="color:rgb(216,57,49);">select * from xxx)</font> as t;</code>

```
			子查询
```

主查询

作用：子查询比较适合复杂查询以及多层级查询结构。

**主查询和子查询的关系:**

1. 子查询是<font style="color:rgb(216,57,49);">嵌入</font>到主查询中
2. 子查询是辅助主查询的，要么充当<font style="color:rgb(216,57,49);">条件</font>，要么充当数据源(<font style="color:rgb(216,57,49);">数据表</font>) => 要么出现在 where，要么出现在 from 位置
3. 子查询是可以<font style="color:rgb(216,57,49);">独立存在</font>的语句，是一条完整的 select 语句

了解：子查询的应用场景

答：在我们需求的基础上，如果这个需求需要通过多条 SQL 语句<font style="color:rgb(216,57,49);">分步查询</font>的情况，一般都需要基于<font style="color:rgb(216,57,49);">子查询</font>。

## 子查询的使用（三步走）

数据集：

```sql
-- 1. 准备数据集
drop table student;

-- 创建student学生表，包含id、name、age、gender、score
create table student(
    id int,
    name varchar(20),
    age int,
    gender enum('male', 'female'),
    score decimal(11,2)
) default charset=utf8mb4;

-- 插入7条测试数据
insert into student values
(1, '张三', 18, 'male', 100.00),
(2, '李四', 19, 'male', 99.00),
(3, '王五', 20, 'male', 98.00),
(4, '赵六', 22, 'male', 97.00),
(5, '钱七', 24, 'male', 96.00),
(6, '孙八', 30, 'male', 95.00),
(7, '周九', 26, 'male', 94.00);
```

**例1. 查询学生表中大于平均年龄的所有学生:**

需求：查询年龄 > 平均年龄的所有学生

前提：① 获取班级的平均年龄值

② 查询表中的所有记录，判断哪个同学 > 平均年龄值

第一步：写子查询

```sql
select avg(age) from students;
```

第二步：写主查询

```sql
select * from students where age > (平均值);
```

第三步：第一步和第二步进行合并

```sql
select * from students where age > (select avg(age) from students);
```

**例2. 查询 goods 产品表中具有分类信息的产品**

数据集：

```sql
create table category(
    cid int,
    cname varchar(20)
) default charset=utf8mb4;

insert into category values
    (1, '手机'),
    (2, '电脑'),
    (3, '图书');

create table goods(
    id int,
    name varchar(20),
    price decimal(11,2),
    cid int
) default charset=utf8mb4;

insert into goods values
    (1, '华为手机', 5000.00, 1),
    (2, '小米手机', 3000.00, 1),
    (3, '苹果手机', 10000.00, 1),
    (4, '华硕电脑', 12000.00, 2),
    (5, '戴尔电脑', 11000.00, 2),
    (6, 'Surface电脑', 13000.00, 2),
    (7, 'Java从入门到放弃',100.00, 3),
    (8, 'HP激光打印机',999.00, 9);
```

需求：查询产品表中具有分类信息的产品（没有与之对应分类信息的产品不显示）

前提：

① 查询分类表中，到底有哪些分类（获取 cid 编号）

② 到产品表中进行判断，判断这个商品的 cid 编号与 ①中的是否相等

第一步：编写子查询

```sql
select cid from category;
```

第二步：编写主查询

```sql
select * from goods where cid in (所有分类cid编号)
```

第三步：把主查询和子查询合并

```sql
select * from goods where cid in (select cid from category);
```

**例3. 查找年龄最小且成绩最高的学生**

注意：<font style="color:rgb(216,57,49);">同时满足以上条件（年龄</font>**<font style="color:rgb(216,57,49);">最小且成绩最高</font>**<font style="color:rgb(216,57,49);">）的数据可能并不存在，此题主要是为了介绍子查询的特殊用法。</font>

第一步：获取年龄最小值和成绩最高值

```sql
select min(age), max(score) from student;
```

第二步：查询所有学员信息（主查询）

```sql
select * from students where (age, score) = (最小年龄, 最高成绩);
```

第三步：把第一步和第二步合并

```sql
select * from student where (age, score) = (select min(age), max(score) from student);
```

子查询是一个<font style="color:rgb(216,57,49);">完整的 SQL 语句</font>，子查询被嵌入到<font style="color:rgb(216,57,49);">一对小括号里面，子查询可以充当条件，也可以充当数据表使用。</font>

掌握子查询编写三步走：<font style="color:rgb(216,57,49);">① 编写子查询 ② 编写主查询并融合伪代码 ③ 伪代码替换为子查询 </font>

## 连接层

主要作用：<font style="color:rgb(216,57,49);">管理和缓冲用户连接，为客户端请求做连接处理；身份认证</font>等。

连接层中的**缓存池**，就是为了<font style="color:rgb(216,57,49);">优化数据库连接而设置的机制，专门用来</font>**<font style="color:rgb(216,57,49);">缓存和复用</font>**<font style="color:rgb(216,57,49);">已建立的连接</font>。其主要作用是：

1. **<font style="color:rgb(216,57,49);">减少连接开销</font>**：避免每次新建连接带来的系统负担，提升性能。
2. **<font style="color:rgb(216,57,49);">提升响应速度</font>**：连接池里有现成的连接，随取随用，减少等待时间。
3. **<font style="color:rgb(216,57,49);">优化资源利用</font>**：通过控制最大连接数，防止过多连接耗尽系统资源。

工作流程很简单：创建连接→用完放回池中→再次复用，保持高效循环。

总结就是：连接池通过<font style="color:rgb(216,57,49);">缓存连接，减少开销、加快响应、合理利用资源，让数据库更高效应对高并发</font>。

连接池配置相关参数：

1. **max\_connections**：指定 MySQL 可以同时处理的最大连接数，控制最大连接数目。
2. **wait\_timeout**：定义一个连接在闲置状态下最多可以等待的时间，超过这个时间将被关闭。
3. **thread\_cache\_size**：控制线程缓存池的大小，以缓存空闲线程，避免频繁创建和销毁线程所带来的开销。

> thread\_cache\_size：根据系统并发情况设置，通常可设置为 **CPU 核心数的2倍**，以便高效处理并发请求

## 引擎层

什么是存储引擎？

* 存储引擎说白了就是 如何<font style="color:rgb(216,57,49);">管理操作数据</font>(存储数据、如何更新、查询数据等)的 一种方法和机制。
* 在 MySQL 数据库中提供了多种存储引擎，<font style="color:rgb(216,57,49);">各个存储引擎的优势各不一样</font> => <code><font style="color:rgb(216,57,49);">show engines</font></code>

常用的存储引擎有哪些？

| 存储引擎 | 描述 |
| --- | --- |
| InnoDB（MySQL5.6版本及以后） | 支持拥有<font style="color:rgb(216,57,49);">ACID特性事务</font>的存储引擎，并且提供<font style="color:rgb(216,57,49);">行级的锁</font>定，支持<font style="color:rgb(216,57,49);">外键</font>、应用广泛。侧重于数据安全，默认引擎 |
| MyISAM（MySQL5.5及之前版本） | <font style="color:rgb(216,57,49);">查询速度快，有较好的索引优化和数据压缩技术</font>；但不支持事务、不支持外键约束。 适用于<font style="color:rgb(216,57,49);">读多写少</font>的应用场景 |
| NDB | 用于MySQL Cluster的集群存储引擎，提供数据层面的高可用性 |
| MEMORY | 存储数据的位置是<font style="color:rgb(216,57,49);">内存</font>，因此<font style="color:rgb(216,57,49);">访问速度最快</font>，但是安全上没有保障。 适合于需要<font style="color:rgb(216,57,49);">快速的访问或临时表</font>。 |
| BLACKHOLE | 黑洞存储引擎，写入的任何数据都会消失，应用于主备复制中的分发主库（中继slave） |

面试：你用过哪些数据库引擎？各自特点？

① MyISAM 引擎，<font style="color:rgb(216,57,49);">擅长数据查询，支持较好的索引优化、数据压缩、支持表级锁以及全文索引技术，安全性相对于 InnoDB 略差一些。</font>

索引优化 => 主键索引（图书目录），有索引，查询速度会更快

数据压缩 => 减少存储空间占用

表级锁 => 只能进行表级锁，就是锁表时，要锁定整个数据表，在这个过程中，这个表只能进行查询操作，而不能进行增删改等操作，但是粒度太大，对并发有一定的影响。

全文索引 => 从一篇文章中搜索指定内容，类似模糊查询，更加强大一些。

② InnoDB 引擎，<font style="color:rgb(216,57,49);">擅长数据安全，支持行级锁，支持事务处理，支持外键约束等等，强调安全性。</font>

行级锁 => 只会对某一行进行锁定，不会全表锁定，粒度更细，并发能力更强。

事务处理 => 一种数据安全策略，保证数据安全

外键约束

③ Memory 引擎，<font style="color:rgb(216,57,49);">擅长数据缓存，加快数据查询，但是由于数据放置于内存，所以安全性没有 MyISAM 以及 InnoDB 好。</font>

前置知识点：索引优化

学不认识的汉字

什么是 explain？

简单来说，`EXPLAIN` 是用来 **查看数据库如何执行一个 SQL 查询** 的工具。它能告诉你数据库在执行查询时，选择了什么样的操作步骤（比如是扫描整个表还是使用索引等），并且通过这些信息你可以判断查询是否高效。

作用：假设你写了一个 SQL 查询来查询数据，如果查询效率低下（比如很慢），你可能想知道数据库是怎么执行的，找出瓶颈在哪。`EXPLAIN` 就是用来帮你“拆解”查询，了解数据库是如何处理每一步的。

如何使用：

```shell
EXPLAIN SELECT * FROM orders WHERE user_id = 123;
```

**分析重点**：

当你在查询前加上 `EXPLAIN` 时，它会显示一个执行计划的表格，这个表格给出了执行查询时的每一个细节。每一行代表 SQL 查询执行的一部分，通常包含这些信息：

* **id**: 这个数字表示查询的执行顺序。对于复杂的查询，可能会有多个查询步骤，`id` 就帮助你理解它们的顺序。
* **select\_type**: 表示查询的类型，是普通查询还是复杂查询（比如涉及子查询等）。
* **table**: 代表查询是从哪张表中获取数据。
* **<font style="color:rgb(216,57,49);">type</font>**<font style="color:rgb(216,57,49);">: 查询是如何连接数据的。这个字段非常重要，它告诉你查询效率的高低，越高效的查询方式类型通常是 </font><code><font style="color:rgb(216,57,49);">ref</font></code><font style="color:rgb(216,57,49);">、</font><code><font style="color:rgb(216,57,49);">range</font></code><font style="color:rgb(216,57,49);"> 等。</font>
* **<font style="color:rgb(216,57,49);">possible\_keys</font>**<font style="color:rgb(216,57,49);">: 查询可能会使用到哪些索引。索引就像数据的“目录”，使用索引可以加速数据查找。</font>
* **<font style="color:rgb(216,57,49);">key</font>**<font style="color:rgb(216,57,49);">: 实际使用了哪个索引。</font>
* **key\_len**: 使用的索引的长度（表示数据库选择了多长的索引）。
* **ref**: 显示与哪个列进行匹配来获取数据。
* **<font style="color:rgb(216,57,49);">rows</font>**<font style="color:rgb(216,57,49);">: 数据库估算的需要扫描的行数。越少的行数，意味着查询越高效。</font>
* **Extra**: 额外信息，比如是否使用了临时表或者文件排序等。

<font style="color:rgb(216,57,49);">核心列 type 说明：</font>

<code><font style="color:rgb(216,57,49);">type</font></code><font style="color:rgb(216,57,49);"> 列非常关键，它反映了查询的效率。这里列出常见的几种类型，效率从高到低排序：</font>

* **<font style="color:rgb(216,57,49);">const</font>**<font style="color:rgb(216,57,49);">：最优，表示数据库通过常量查找一个行。</font>
* **<font style="color:rgb(216,57,49);">ref</font>**<font style="color:rgb(216,57,49);">：比较高效，表示数据库通过索引查找匹配行。</font>
* **<font style="color:rgb(216,57,49);">range</font>**<font style="color:rgb(216,57,49);">：表示数据库通过范围扫描索引来查找数据。</font>
* **<font style="color:rgb(216,57,49);">index</font>**<font style="color:rgb(216,57,49);">：表示数据库扫描整个索引（但不会读取数据表），效率不如 </font><code><font style="color:rgb(216,57,49);">range</font></code><font style="color:rgb(216,57,49);">。</font>

<font style="color:rgb(216,57,49);">索引=字段目录，数据=记录</font>

* **<font style="color:rgb(216,57,49);">ALL</font>**<font style="color:rgb(216,57,49);">：最差，表示数据库执行了全表扫描，效率最低。</font>

**总结对比**

| **<font style="color:rgb(0, 0, 0);">类型</font>** | **<font style="color:rgb(0, 0, 0);">类比</font>** | **<font style="color:rgb(0, 0, 0);">速度</font>** | **<font style="color:rgb(0, 0, 0);">适用场景</font>** |
| :---: | --- | :---: | :---: |
| <font style="color:#f54a45;">const</font> | <font style="color:rgb(0, 0, 0);">直接翻到指定页码</font> | <font style="color:rgb(0, 0, 0);">最快</font> | <font style="color:#f54a45;">主键/唯一索引的等值查询</font> |
| <font style="color:#f54a45;">ref</font> | <font style="color:rgb(0, 0, 0);">按目录查几个名字</font> | <font style="color:rgb(0, 0, 0);">快</font> | <font style="color:#f54a45;">普通索引的等值查询</font> |
| <font style="color:#f54a45;">range</font> | <font style="color:rgb(0, 0, 0);">按目录查一个连续区间（如 20~30 岁）</font> | <font style="color:rgb(0, 0, 0);">较快</font> | <font style="color:#f54a45;">索引范围查询</font> |
| <font style="color:#f54a45;">index</font> | <font style="color:rgb(0, 0, 0);">扫完整本目录（不读正文）</font> | <font style="color:rgb(0, 0, 0);">较慢</font> | <font style="color:rgb(0, 0, 0);">只需索引字段，不查数据</font> |

<font style="color:rgb(216,57,49);">面试题：在运维环境中，发现 MySQL 运行缓慢，如何去解决？</font>

<font style="color:rgb(216,57,49);">答：</font>

<font style="color:rgb(216,57,49);">① 从系统层面，检查系统资源使用情况，如 CPU 负载（top）、检查内存占用（free -h）、检查磁盘空间使用（df -h），因为 mysql 把数据写入到磁盘，读写都涉及磁盘 io（iostat）</font>

<font style="color:rgb(216,57,49);">② 从日志层面，开启慢查询日志，把查询缓慢的 SQL 写入到慢查询日志中，进行捕获</font>

<font style="color:rgb(216,57,49);">③ 从 SQL 语句层面，使用 explain 执行计划分析 SQL 执行过程，是否有走缓存等等，查看具体慢的原因。如果是全表扫描，可以考虑引入索引（如主键索引、唯一索引、普通索引）等等实现优化操作</font>

## 实时监控 SQL 查询

**作用**：`SHOW PROCESSLIST`可以显示<font style="color:rgb(216,57,49);">当前正在执行的 SQL 语句及其状态，帮助你发现正在运行的长时间查询</font>。

注意：适合执行时间较长的 SQL 语句

```sql
SHOW PROCESSLIST;
```

分析重点：

**Command**：表示查询的状态，比如<code><font style="color:rgb(216,57,49);">Query</font></code><font style="color:rgb(216,57,49);">表示正在执行查询</font>，<code><font style="color:rgb(216,57,49);">Sleep</font></code><font style="color:rgb(216,57,49);">表示连接空闲</font>。

**Time**：表示<font style="color:rgb(216,57,49);">查询的执行时间，时间较长的查询可能是性能瓶颈</font>。

**State**：显示当前查询执行的阶段，帮助<font style="color:rgb(216,57,49);">定位查询的卡顿点</font>（如`Copying to tmp table`表示查询在使用临时表）。

> 通过上面实时监控的操作，我们可以监测到目前 MySQL 数据库中是否有正在执行的 sql 查询操作。

## 使用<code><font style="color:rgb(216,57,49);">SHOW PROFILES</font></code>查看查询的详细耗时（重点）

**作用**：`SHOW PROFILES`可以显示最近<font style="color:rgb(216,57,49);">执行的 SQL 查询的执行时间，以及每个阶段的耗时，帮助你了解查询在执行时哪个环节耗时最长。</font>

```sql
SET profiling = 1;                                         -- 开启性能分析（开关）
alter table simple_table change id id int;
alter table simple_table drop primary key;
select * from simple_table where id = 1990000;

SHOW PROFILES;                                             -- 查看所有SQL的执行情况
SHOW PROFILE FOR QUERY 1;                                  -- 查看第一个查询的详细耗时
```

小结：

如果我们想捕获 SQL 语句详细执行以及底层执行过程以及耗时都可以使用(show profiles;)
