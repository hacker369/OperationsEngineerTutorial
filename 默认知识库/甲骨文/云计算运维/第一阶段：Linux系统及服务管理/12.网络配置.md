## Linux网络接口名称规则

```shell
Network interface names

Traditionally, network interfaces in Linux are enumerated as eth0, eth1, eth2, and so on. However, the mechanism which 
sets these names can cause changes to which interface gets which name as devices are added and removed. 
The default naming behavior in Red Hat Enterprise Linux 7 is to assign fixed names based on firmware, device topology, 
and device type. 

Interface names have the following characters:
Ethernet interfaces begin with en, WLAN interfaces begin with wl, and WWAN interfaces begin with ww.
The next character(s) represents the type of adapter with an o for on-board, s for hotplug slot, and p for PCI geographic location. 
Not used by default but also available to administrators, an x is used to incorporate a MAC address.
Finally, a number N is used to represent an index, ID, or port.
If the fixed name cannot be determined, the traditional names such as ethN will be used. 

For example, the first embedded network interface may be named eno1 and a PCI card network interface may be named enp2s0. 
The new names make it easier to distinguish the relationship between a port and its name if the user knows both, but the
trade off is that users cannot assume a system with one interface calls that interface eth0. 
```

```shell
en:ethernet 以太网-双绞线
wl:wirelessLAN：无线网卡
ww：WAN广域网：串行线缆

o：on-board板载网卡
s:hotplug热插拔
p：PCI接口

N：序号或者ID

示例：ens33
```

## CentOS9配置IP

CentOS Stream 9 的网卡配置文件在`/etc/NetworkManager/system-connections/`目录下，打开对应的网卡配置文件。（说明：我们现在的Linux其实是可以上网的，但是目前使用的IP是自动获取的，我们设置为手动指定的IP！）

**网卡配置**（注意自己的网卡名称）

```shell
打开网络配置文件
# vim /etc/NetworkManager/system-connections/ens33.nmconnection
先将你的内容全部删除（dG或400dd）,然后复制以下内容到你的文件中
[connection]
id=ens18
uuid=8d1ece55-d999-3c97-866b-d2e23832a324
type=ethernet
autoconnect-priority=-999
interface-name=ens18
permissions=
timestamp=1639473429

[ethernet]
mac-address-blacklist=

[ipv4]
address1=192.168.1.92/24,192.168.1.1
dns=8.8.8.8;
dns-search=
method=manual

[ipv6]
addr-gen-mode=eui64
dns-search=
method=auto

[proxy]
```

**将上面的内容复制到自己的Linux服务器中，做如下修改：**

* id改成自己网卡的名字

## nmcli配置IP

上面是通过配置文件的方式来配置IP，其实我们也可以通过命令的方式来配置IP。使用`nmcli`命令即可。其实我们通过`nmcli`命令配置好的IP最后也就写入到了上面的**配置文件**中了！

将 IPv4 地址 (192.168.126.180) 调配给 ens33 网卡上（IP地址不能随便配，查看自己Linux中动态获取的IP是多少，稍微改变最后一位即可）

```shell
# nmcli connection modify ens33 ipv4.addresses 192.168.126.180/24
```

nmcli 命令设置网关

```shell
# nmcli connection modify ens33 ipv4.gateway 192.168.126.2
```

设置手动配置（从 dhcp 到 static）

```shell
# nmcli connection modify ens33 ipv4.method manual
```

设置 DNS 值为 “8.8.8.8”（谷歌的域名解析器）

```shell
# nmcli connection modify ens33 ipv4.dns "114.114.114.114,8.8.8.8"
```

保留上述更改并从新加载

```shell
# nmcli connection up ens33
```

#### 端口号

是一段数字：0-65535

在Linux系统中并非所有的程序启动后都会占用端口，端口号的使用与程序是否涉及网络通信直接相关。

如果该程序需要通过TCP/UDP协议进行网络数据传输（如浏览器、web服务器、FTP客户端等），则会占用端口号以标识自身。

通过检查端口号，可以确认服务是否正确启动

#### Show TCP sockets

什么端口不重要

```shell
# ss -tnl
State Recv-Q Send-Q Local Address:Port Peer Address:Port 
LISTEN 0 128 *:80 *:* 

# ss -tna

# ss -tnlp

说明：
ss：Socket Statistics，是用于显示活动套接字信息的工具
-t：仅显示TCP连接‌
-n：禁用服务名称解析（显示数字端口）‌
-l：显示处于监听状态的套接字
-a：显示所有连接（包括监听和非监听）

State	连接状态
  LISTEN：套接字正在监听连接请求（服务端等待客户端连接）‌
  ESTABLISHED：已建立有效连接，双方可正常通信‌
Recv-Q	接收队列待处理数据量‌
Send-Q	发送队列待处理数据量‌
Local Address:Port	本地IP和端口
Peer Address:Port ‌	表示与当前主机建立通信的‌远程设备IP地址和端口号

也可以用这个查看端口号的占用情况
# netstat -tnlp
-t：显示tcp连接的
-n：以数字的形式显示端口
-l：显示处于listen 监听状态的
-p：显示process进程相关的信息
```

重要的是要知道端口的概念和查看的方法

## 配置IP地址

将最小化安装的系统设置为手动指定的IP。

```shell
# nmcli connection modify ens33 ipv4.addresses 192.168.126.180/24
```

nmcli 命令设置网关

```shell
# nmcli connection modify ens33 ipv4.gateway 192.168.126.2
```

设置手动配置（从 dhcp 到 static）

```shell
# nmcli connection modify ens33 ipv4.method manual
```

设置 DNS 值为 “8.8.8.8”（谷歌的域名解析器）

```shell
# nmcli connection modify ens33 ipv4.dns "114.114.114.114,8.8.8.8"
```

保留上述更改并从新加载

```shell
# nmcli connection up ens33
```

## 同步日期时间

查看chrony运行状态：

```shell
# systemctl status chronyd
```

![1753699076143-e4131670-b621-48b3-b446-c6e08c1e2de1.png](./img/F1N6pfgO4NcUoj3_/1753699076143-e4131670-b621-48b3-b446-c6e08c1e2de1-496198.png)

如果chrony服务没有启动或者没有设置为开机自启，就执行下面的命令：

```shell
# systemctl start chronyd
# systemctl enable chronyd
```

配置NTP时间服务器，默认配置文件在`/etc/chrony.conf`

```shell
# vim /etc/chrony.conf
server ntp.aliyun.com iburst
server ntp1.tencent.com iburst
```

重启chrony服务：

```shell
# systemctl restart chronyd
```
