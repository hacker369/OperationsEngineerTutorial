# 10.计划任务

# 一、简介

## 作用

计划任务主要是做一些周期性的任务，目前最主要的用途是定期备份数据。

## 分类

一次性调度执行 at，Schedule one-time tasks with at.

循环调度执行 cron，Schedule recurring jobs with cron.

# 二、一次性调度执行 at

## 语法格式

语法格式：

```shell
# at <TIMESPEC>
```

<TIMESPEC>示例：

```shell
now +5min															5分钟后
teatime tomorrow (teatime is 16:00)		下午茶时间
noon +4 days													第四天中午
5pm august 3 2024											2024年8月3日下午5点
4:00 2024-11-27												某年某月某日某时
```

## 案例

初识一次性任务计划

1.设置一个定时创建用户的任务

```shell
# at now +1min
at> useradd uuuu
at> <EOT>				CTRL+D输入完毕，提交任务
job 1 at Sat Mar 21 22:34:00 2023
```

2.查询任务

```shell
# atq
3 Thu Mar 30 09:55:00 2023 a root

说明：
atq 命令用来查询待执行任务的信息，显示的字段为：
作业号 执行时间 队列名称 用户
atq | grep 用户名，可以查询指定用户的任务
```

3.验证结果

```shell
# id uuuu
查出用户信息即可
```

4.报错

```shell
如果提示如下错误：Can't open /var/run/atd.pid to signal .No atd running?
说明系统未启动at程序
手动启动即可：
# systemctl start atd
```

# 三、循环调度执行 cron

## 简介

* cron的概念和crontab是不可分割的。
* crontab是一个命令，常见于Unix和Linux的操作系统之中
* crontab命令是用于设置周期性被执行的指令

## 查看进程状态

默认情况下，cron定时任务是在后台处于开启状态，而且是开机自启的，我们可以查看它的进程状态。

```shell
# systemctl status crond
# ps aux | grep crond
root 550 0.0 0.0 126300 1648 ? Ss 10:05 0:00 /usr/sbin/crond -n
crond程序运行是计划任务执行的根本

ps命令说明：
ps：process status，进程状态，用于显示当前进程信息
-a：显示所有用户的进程
-u：以用户友好的格式显示详细信息（如CPU、内存占用等）
-x：显示没有控制终端的进程（如后台服务、守护进程）
```

## cron示例

### 管理方式

#### 创建计划

```shell
# crontab -e     Edit jobs for the current user
```

#### 查询计划

```shell
# crontab -l    List the jobs for the current user
管理员可以使用 -u username, 去管理其他用户的计划任务，比如：
# crontab -u zhangsan -l	列出zhangsan用户的计划任务
# crontab -u zhangsan -e 	编辑zhangsan用户的计划任务
# crontab -u zhangsan -r 	删除zhangsan用户的任务计划
```

#### 删除计划

```shell
# crontab -r     Remove all jobs for the current users.
```

### 计划任务存储位置

```shell
# ls /var/spool/cron/<用户名>
空的说明没有任务
```

### 语法格式 Job format

```shell
# .---------------- minute (0 - 59)
# | .------------- hour (0 - 23)
# | | .---------- day of month (1 - 31)
# | | | .------- month (1 - 12) OR jan,feb,mar,apr ...
# | | | | .---- day of week (0 - 6) (Sunday=0 or 7) OR sun,mon,tue,wed,thu,fri,sat
# | | | | |
# * * * * * command

说明:
分 时 日 月 周     命令或脚本程序
六个部分用空格隔开
```

示例1：

```shell
0 * * * * /mysql_back.sh				每小时的0分执行
*/5 * * * * /mysql_back.sh			每五分钟执行
0 2 1,4,6 * * /mysql_back.sh		每月的1,4,6日的2点整执行
0 2 5-9 * * /mysql_back.sh			每月5日到9日的两点整执行
* * * * * /mysql_back.sh				每分钟执行
0 2 * * * /mysql_back.sh				每天两点整执行
0 2 14 * * /mysql_back.sh				每月14号的2点整执行
0 2 14 2 * /mysql_back.sh				2月14日2点整执行
0 2 * * 5 /mysql_back.sh				不写月日，仅周生效，每月的周五，两点执行
0 2 2 6 5 /mysql_back.sh				书写月和日，月或日，月日周均生效，6月2日的两点整执行，6月的周五两点整执行
```

示例2：

```shell
00 02 * * * ls									每天2:00整
00 02 1 * * ls									每月1号2:00整
00 02 14 2 * ls									每年2月14号2:00整
00 02 * * 7 ls									每周日2:00整
00 02 * 6 5 ls									每年6月的周五2:00整  （特殊）
00 02 14 * 7 ls									每月14号2:00整，每周日2:00整，这两个时间都执行
00 02 14 2 7 ls									每年2月14号2:00整，2月每周日2:00整，这两个时间都执行
```

### 示例1

#### 需求

每隔1分钟执行一次date命令，获取当前系统的日期时间。

#### 实现

```shell
# 创建任务计划
# crontab -e
* * * * * /usr/bin/date		（一般我们在任务计划中写的命令都是写完整路径！！！）

# 查看任务计划
# crontab -l
```

#### 效果

上面定时任务执行的效果是在终端看不出来的，需要查看定时任务的日志看结果：`cat /var/log/cron`

> 扩展：
>
> `tail -20 文件路径`可以查看一个文件的后20行内容
>
> `tail -f 文件路径`实时看文件的最新的内容，监控这个文件

### 示例2

#### 需求

每隔一分钟执行创建文件的计划任务

#### 准备创建脚本文件

```shell
# vim /root/1.sh
输入以下内容
/usr/bin/touch /root/`date +%F-%H-%M-%S`.txt

# chmod +x /root/1.sh

说明：
.sh 						结尾的是shell脚本文件，后面我们会详细讲解
/usr/bin/touch	是touch命令的完整路径写法
date						是查看当前日期时间的命令，%Y表示四位数的年份，%m表示两位的月份，%d表示两位的日期，%H表示24小时制小时，%M表示分钟，%S表示秒，%F表示完整的日期等价于%Y-%m-%d
``							反单引号，用它引起来的命令会先执行
shell脚本文件要执行，需要赋予执行权限，所以有 chmod +x /root/1.sh
```

#### 编写任务计划

```shell
# crontab -e
输入以下内容
* * * * * /root/1.sh

# 查看计划任务
# crontab -l
```

#### 查看创建结果

```shell
# ls -l /root/
```

> 看到结果后，我们将该计划任务删除，使用`crontab -r`命令

## 任务执行日志

计划任务执行完后，系统会记录相关日志信息。

计划任务的日志文件：`/var/log/cron`。
