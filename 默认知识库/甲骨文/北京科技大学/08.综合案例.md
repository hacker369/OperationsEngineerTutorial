# 四、实现条件查询功能
## 编写 mapper 层代码
```java
package com.lhp.mapper;

import com.lhp.pojo.Emp;
import org.apache.ibatis.annotations.Param;
import java.util.List;


public interface EmpMapper {

    List<Emp> findEmp(@Param("name") String name, @Param("dept") String dept);
}
```

```xml
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.lhp.mapper.EmpMapper">

    <select id="findEmp" resultType="com.lhp.pojo.Emp">
        select * from emp
        <where>
            <if test="name != null and name != ''">
                name like concat('%', #{name}, '%')
            </if>
            <if test="dept != null and dept != ''">
                and dept = #{dept}   
            </if>
        </where>
    </select>
</mapper>
```

package com.lhp.service.impl;

import com.github.pagehelper.PageHelper;
import com.github.pagehelper.PageInfo;
import com.lhp.mapper.EmpMapper;
import com.lhp.pojo.Emp;
import com.lhp.service.EmpService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import java.util.List;


@Service
public class EmpServiceImpl implements EmpService {

    @Autowired
    private EmpMapper empMapper;

    @Override
    public void deleteEmp(Integer id) {
        empMapper.deleteEmp(id);
    }

    @Override
    public void updateEmp(Emp emp) {
        empMapper.updateEmp(emp);
    }

    @Override
    public void addEmp(Emp emp) {
        empMapper.addEmp(emp);
    }

    @Override
    public PageInfo<Emp> findEmp(String name, String dept, Integer pageNum, Integer pageSize) {
        PageHelper.startPage(pageNum, pageSize);
        List<Emp> emp = empMapper.findEmp(name, dept);
        PageInfo<Emp> pageInfo = new PageInfo<Emp>(emp);
        return pageInfo;
    }
}

public class EmpController {

    @Autowired
    private EmpService empService;

    @GetMapping("/exportEmp")
    public String exportEmp(HttpServletResponse response) throws IOException {

        PageInfo<Emp> pageInfo = empService.findEmp(null, null, 1, 1000);

        String fileName = "D:/员工信息-" + System.currentTimeMillis() + ".xlsx";
        // 这里 需要指定写用哪个class去写，然后写到第一个sheet，名字为模板 然后文件流会自动关闭
        // 如果这里想使用03 则 传入excelType参数即可
        EasyExcel.write(fileName, Emp.class)
                .sheet("员工信息")
                .doWrite(() -> {
                    // 分页查询数据
                    return pageInfo.getList();
                });
        return fileName;
    }

@RequestMapping("/emp")
public class EmpController {

    @Autowired
    private EmpService empService;

    @PostMapping("/importEmp")
    public void importEmp(MultipartFile file) throws IOException {
        List<List<Object>> list = ExcelUtil.getReader(file.getInputStream(), 0).read(1);
        List<Emp> emps = new ArrayList<>();
        for (List<Object> o : list) {
            Emp emp = new Emp();
            emp.setName(o.get(0).toString());
            emp.setSex(o.get(1).toString());
            emp.setAge(Integer.parseInt(o.get(2).toString()));
            emp.setAddress(o.get(3).toString());
            emp.setBirthday(DateUtil.parse(o.get(4).toString(), "yyyy-MM-dd"));
            emp.setSalary(Double.parseDouble(o.get(5).toString()));
            emp.setDept(o.get(6).toString());
            emp.setPosition(o.get(7).toString());
            emps.add(emp);
        }
        empService.imortEmp(emps);
    }

# 十三、表单校验--扩展
## 说明
目前，我们在添加一个员工时，如果用户没有输入员工的名字、生日等信息，直接就点击了保存按钮，则会给数据库添加一条空信息的数据！不合适！

我们应该在用户点击保存按钮的时候，对一些员工信息进行校验，比如：名字必须输入、生日必须选择一个日期、部门必须选择一个等等！

解决办法：

可以通过 ElementUI 中提供的表单校验来解决！

# 十四、Git 的使用
## 团队开发
在实际工作中，我们都是一个团队在完成项目的开发，每个成员负责项目中的一部分模块。每个人将自己的模块开发完后，需要将团队所有成员的代码给整合起来。

在这个过程中，会涉及到如下**问题：**

1. 什么时候进行代码的整合

2. 怎样进行代码的整合，并且整合后怎样将代码同步给所有团队成员

3. 怎样有选择性的将我们本地部分开发好的功能和其他成员的代码整合起来，并且如果整合后有问题的话，还可以回退到之前的状态

解决：

**我们就是需要一款管理代码的软件 + 能够存储代码的公共空间。**

该软件具备如下特点：

1. 管理本地代码

2. 管理公共空间的代码

3. 能够实现代码的整合和回退

那这款工具其实就是我们要学习的版本控制工具。常见的有：SVN 和 <font style="background-color:rgb(255,255,0);">Git </font>等。

## **<font style="color:rgb(0,0,0);">Git 介绍</font>**
Git 是一个免费的、开源的分布式版本控制系统，可以快速高效地处理从小型到大型的项目。

本质：项目开发的管理工具，使用 Git 可以方便的完成团队的协作开发。以及项目开发过程中的代码管理。实现团队协作时候的资源共享。

## 存储流程
![1719823798486-5d3f1e77-ff98-4ed7-bd38-4c1c7407b55e.png](./img/AwArEq4fB9Pln2vi/1719823798486-5d3f1e77-ff98-4ed7-bd38-4c1c7407b55e-411213.png)

1. 工作区、暂存区和本地仓库，说的其实都是我们本地电脑。

2. 我们可以在工作区创建文件写代码，写完后，该文件处于已修改状态（modified）。已修改状态就表明文件进行了修改，但还没有提交保存。

3. 使用命令git add 可以将处于 modified 状态的文件添加到暂存区，这时文件处于已暂存状态（staged）。已暂存状态就表示将已修改的文件放到了暂存区，下次提交代码时要提交它。

4. 使用命令 git commit 可以将暂存区的文件提交到本地仓库，这时文件处于已提交状态（commited）。已提交状态就表示该文件已经被保存在了本地仓库中，并且生成一个新的版本。
