# 06.管道命令详解

## <font style="color:rgb(51, 51, 51);">管道符</font>
<font style="color:rgb(51, 51, 51);">"|"，竖线</font>

<font style="color:rgb(51, 51, 51);">作用：管道是一种通信机制，通常用于进程间的通信。它表现出来的形式将前面每一个进程的输出（stdout）直接作为下一个进程的输入（stdin）。</font>

![1737534276034-e275a25e-3b1a-4b04-9f8d-bbca47469a46.png](./img/Ljy7ZcHfqTw7H1at/1737534276034-e275a25e-3b1a-4b04-9f8d-bbca47469a46-425583.png)

<font style="color:rgb(51, 51, 51);">0 ：标准输入，程序或命令需要外部的某些程序传递相应的参数，才能正常运行</font>

<font style="color:rgb(51, 51, 51);">1 ：标准输出，程序或命令正确的执行结果，我们就称之为标准输出</font>

<font style="color:rgb(51, 51, 51);">2 ：标准错误，程序或命令错误的执行结果，我们就称之为标准错误</font>

## <font style="color:rgb(51, 51, 51);">过滤（筛选）功能</font>
<font style="color:rgb(51, 51, 51);">基本语法：</font>

```shell
# 前一个命令 | 后一个命令
```

<font style="color:rgb(51, 51, 51);">案例：获取/根目录下包含关键字"y"的文件信息</font>

```shell
# ls / | grep y
解析：
| ：管道符号，作用=>把ls /的正确的执行结果作为参数传递给grep命令
ls / ：管道左边的命令
grep y ：管道右边的命令
```

> <font style="color:rgb(119, 119, 119);">grep命令的基本语法 => grep 关键字 文件名称</font>
>

<font style="color:rgb(51, 51, 51);">案例：检索系统中的已安装文件，只筛选mariadb软件信息</font>

```shell
# rpm -qa | grep mariadb
-q ：query，查询
-a ：all，所有
```

<font style="color:rgb(51, 51, 51);">案例：在系统的进程中进行查找，查找与root相关的进程信息</font>

```shell
# ps -ef | grep root
ps -ef ：查询系统中所有正在运行的进程
```

## <font style="color:rgb(51, 51, 51);">特殊功能</font>
<font style="color:rgb(51, 51, 51);">通过管道的操作方法来实现less 的等价效果（了解）</font>

<font style="color:rgb(51, 51, 51);">例如：源指令是"less xxx.txt"，使用管道的话则可以写成：cat xxx.txt | less</font>

```shell
# less initial-setup-ks.cfg
更改为管道命令
# cat initial-setup-ks.cfg | less
```

## <font style="color:rgb(51, 51, 51);">统计功能（重点）</font>
<font style="color:rgb(51, 51, 51);">回顾：wc命令，word count，文件统计功能</font>

```shell
# wc [选项] 文件名称
选项说明：
-l : 统计总行数
-w : word，总单词数
-c : 统计总字节数
```

<font style="color:rgb(51, 51, 51);">案例：统计/根目录下一共有多少个文件</font>

```shell
# ls / | wc -l
```

<font style="color:rgb(51, 51, 51);">案例：用户在计算机中有一个配置文件（/etc/passwd），一般情况下，一个用户会占用一行配置，请你使用现学的管道统计当前计算机中一共有多少个用户个信息（一个用户一行）</font>

```shell
# cat /etc/passwd | wc -l
```

## <font style="color:rgb(51, 51, 51);">xargs命令扩展</font>
<font style="color:rgb(51, 51, 51);">问题：为什么需要xargs命令？</font>

<font style="color:rgb(51, 51, 51);">答：之所以能用到这个命令，关键是由于很多命令不支持 | 管道来传递参数，而日常工作中又有这个必要，所以就有了 xargs 命令。</font>

> <font style="color:rgb(119, 119, 119);">简单来说，xargs命令就相当于对管道命令进行了一个扩展，让所有命令都支持管道</font>
>

<font style="color:rgb(51, 51, 51);">案例：搜索/etc目录下的所有".conf"结尾的文件信息，然后以详细列表形式显示</font>

```shell
# find /etc -name "*.conf" | ls -l
```

<font style="color:rgb(51, 51, 51);">解决方案，在ls命令之前添加一个xargs命令，这样ls命令就支持管道了，可以用于接收前一个命令的执行结果</font>

```shell
# find /etc -name "*.conf" | xargs ls -l
```
