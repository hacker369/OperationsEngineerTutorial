# 14.磁盘管理（下）

# 一、逻辑卷管理

## <font style="color:rgb(0,0,0);">内容回顾</font>

之前我们学习过fdisk和parted进行分区，这种分区有一些不好的地方。

比如：我们的一块硬盘分了3个分区，/分区，/boot分区和swap分区。我们将数据都存储在/分区中，如果/分区满了，那我们如果给服务器加一块硬盘，这块硬盘的空间是不能直接给加到之前分好的根分区的！

只能是将新硬盘再分区，然后改变我们写好的一些应用程序代码，将数据写入到新的分区中。但是这个工作量很大，而且不好搞！

这就需要用到LVM（逻辑卷管理）了，使用LVM就可以将新硬盘的空间加入到之前分好的根分区中了！可以实现动态扩容！

以后我们在工作中进行分区的话，也是使用LVM，而不是用之前的方式！

## <font style="color:rgb(0,0,0);">LVM概述</font>

<font style="color:rgb(0,0,0);">LVM 是 Logical Volume Manager 的简称，中文就是逻辑卷管理。</font>

![1740664716457-2c0426ef-e2f8-47c0-832a-4717c385a92b.png](./img/WMCYsYynTN39nIel/1740664716457-2c0426ef-e2f8-47c0-832a-4717c385a92b-211658.png)

* <font style="color:rgb(0,0,0);">物理卷（PV，Physical Volume）：就是真正的物理硬盘或由物理硬盘分成的分区。</font>
* <font style="color:rgb(0,0,0);">卷组（VG，Volume Group）：将多个物理卷合起来就组成了卷组，组成同一个卷组的物理卷可以是同一个硬盘的不同分区，也可以是不同硬盘上的不同分区。我们可以把卷组想象为一个逻辑硬盘，它是由多个硬盘的多个分区组成。</font>
* <font style="color:rgb(0,0,0);">逻辑卷（LV，Logical Volume）：卷组是一个逻辑硬盘，硬盘必须分区之后才能使用，这个分区我们称作逻辑卷。逻辑卷可以格式化和写入数据。我们可以把逻辑卷想象成为分区。</font>
* <font style="color:rgb(0,0,0);">物理扩展（PE，Physical Extend）：PE 是用来保存数据的最小单元，我们的数据实际上都是写入PE当中，PE 的大小是可以配置的，默认是 4MB。（之前我们做分区的时候，设置的分区大小其实都转换为了占用多少柱面，这里的PE也是一个道理，分成多大的逻辑卷，其实就是转换为了占用多少个PE）</font>

> 注意：/boot分区是不能使用LVM的，必须使用之前的物理分区。其他的分区可以使用LVM。

## 建立LVM的步骤

1. <font style="color:rgb(0,0,0);">首先需要把物理硬盘分成分区，当然也可以是整块物理硬盘。 </font>
2. <font style="color:rgb(0,0,0);">然后把物理分区建立成为物理卷（PV），也可以直接把整块硬盘都建立为物理卷。</font>
3. <font style="color:rgb(0,0,0);">接下来把物理卷整合成为卷组（VG）。卷组就已经可以动态的调整大小了，可以把物理分区加入卷组，也可以把物理分区从卷组中删除。</font>
4. <font style="color:rgb(0,0,0);">最后就是把卷组再划分成为逻辑卷（LV），当然逻辑卷也是可以直接调整大小的。我们说逻辑卷可以想象成为分区，所以也需要格式化和挂载。</font>

将物理分区或者硬盘建立为物理卷的命令：

```shell

```

### 第五步：创建卷组

创建卷组（VG）使用的命令是：

```shell
# vgcreate [选项] 卷组名 物理卷名1 物理卷名2 ...

选项说明：
-s PE 大小：指定 PE 的大小，单位可以是 MB,GB,TB 等。如果不写默认 PE 大小是 4MB，我们一般使用默认即可。
```

我们目前有3个物理卷，本次实验中，我们先将两个物理卷加入到卷组中，另外一个后面我们演示动态扩容，然后再将最后一个物理卷加入到卷组中。

实际工作中，如果卷组的空间用完了，我们可以给服务器新加一块硬盘，然后将这个硬盘直接创建为一个物理卷，然后再将物理卷加入到卷组中即可实现卷组的动态扩容！

```shell
# 创建一个卷组，名字为testvg，将/dev/sdb5和/dev/sdb6这两个物理卷加入到卷组中
# vgcreate testvg /dev/sdb5 /dev/sdb6
  Volume group "testvg" successfully created
```

### 扩展操作：扩容卷组

本步骤可以不做！（在本次实验中我做了这一步）

我们这里只是给大家演示一下，如果以后卷组空间不够了，我们可以将一个物理分区创建为物理卷，然后将这个物理卷加入到卷组中，从而实现动态扩容卷组。

当然了，也可以将一个新的硬盘创建为物理卷，然后将物理卷加入到卷组中，实现卷组的动态扩容。

本次实验，我们将上面创建好的物理卷`/dev/sdb7`加入到卷组`testvg`中，实现卷组的动态扩容。

```shell
# 将/dev/sdb7这个物理卷加入到testvg这个卷组中
# vgextend testvg /dev/sdb7
  Volume group "testvg" successfully extended
```

### 扩展操作：减少卷组容量

本操作不建议做，一般我们都是扩容卷组，不会减少卷组的。减少卷组可能丢失数据。

```shell
# 将/dev/sdb7这个物理卷从testvg这个卷组中移除
# vgreduce testvg /dev/sdb7 
 Removed "/dev/sdb7" from volume group "scvg"
```

```shell
# 删除卷组中所有未使用的物理卷
# vgreduce -a
```

### 扩展操作：删除卷组

```shell
# 将testvg这个卷组删除
# vgremove testvg

```

<font style="color:rgb(0,0,0);">卷组删除之后，才能删除删除物理卷。还要注意的是 testvg 卷组还没有添加任何的逻辑卷，那如果拥有了逻辑卷，记得先删除逻辑卷再删除卷组。删除就是安装的反过程，每一步都不能跳过。</font>

### <font style="color:rgb(0,0,0);">第七步：创建逻辑卷</font>

目前我们的卷组是6G的大小，在这个卷组中我们可以随意划分指定逻辑卷的大小！

创建逻辑卷的语法：

```shell
# lvcreate [选项] 卷组名

选项说明：
-L：指定逻辑卷的大小，单位是MB、GB、TB等
-l：按照PE个数指定逻辑卷的大小，这个参数需要换算容量，不方便
-n：指定逻辑卷名称
```

本次案例操作：

```shell
# 从卷组testvg中创建一个名称为testlv的逻辑卷，逻辑卷的大小为2G
# lvcreate -L 2G -n testlv testvg
```

查看逻辑卷信息：

```shell
# lvdisplay
```

### 第八步：格式化

建立完逻辑卷后，还是需要进行格式化操作的。跟我们之前的物理分区一样，分区完后需要对分区进行格式化操作。

逻辑卷格式化的命令和之前的基本分区是一样的，不过逻辑卷的设备名是：`/dev/卷组名/逻辑卷名`

本次实验操作：

```shell
# 格式化逻辑卷
# mkfs.xfs /dev/testvg/testlv
meta-data=/dev/testvg/testlv     isize=512    agcount=4, agsize=131072 blks
         =                       sectsz=512   attr=2, projid32bit=1
         =                       crc=1        finobt=0, sparse=0
data     =                       bsize=4096   blocks=524288, imaxpct=25
         =                       sunit=0      swidth=0 blks
naming   =version 2              bsize=4096   ascii-ci=0 ftype=1
log      =internal log           bsize=4096   blocks=2560, version=2
         =                       sectsz=512   sunit=0 blks, lazy-count=1
realtime =none                   extsz=4096   blocks=0, rtextents=0
```

### 第九步：挂载逻辑卷

逻辑卷也是和之前的分区一样，需要进行挂载。命令也都一样。

```shell
# 创建挂载点
# mkdir /mnt/lv_disk

# 挂载逻辑卷
# mount /dev/testvg/testlv /mnt/lv_disk/

# 查看挂载情况
# mount |grep "/dev"

# 或者
# df -hT
```

本次挂载完，也是临时挂载。如果要永久挂载，和之前的基本分区一样，修改`/etc/fstab`配置文件即可。

### 第十步：调整逻辑卷的大小

我们可以调整逻辑卷的大小，只要卷组中有剩余空间。而且调整逻辑卷的大小不会影响到当前逻辑卷中的数据！

当然了，如果卷组中空间不够，我们也可以加一块硬盘，将硬盘分区或者不分区也行，将分区或者硬盘变为物理卷，然后将物理卷加入到卷组中，然后再给逻辑卷扩容！

注意：我们可以将逻辑卷调大，也可以将其调小，一般我们都调大，不会调小！

调整逻辑卷大小：

```shell
# lvresize [选项] 逻辑卷设备文件名

选项说明：
-L：逻辑卷容量大小调整，单位是KB、GB、TB等。使用+代表增加空间，-代表减少空间，如果直接写容量，代表设置逻辑卷大小为指定的大小。
-l：按照PE个数调整逻辑卷大小
```

之前我们的逻辑卷大小为2G，卷组中还剩余4G的空间，现在我们将逻辑卷大小调整为3G。

```shell
# 调整逻辑卷的大小为3G
# lvresize -L 3G /dev/testvg/testlv

# 查看逻辑卷的信息
# lvdisplay
```

### 扩展操作：删除逻辑卷

删除逻辑卷之前，首先需要卸载。删除逻辑卷语法：

```shell
# lvremove 逻辑卷设备文件名
```

具体操作：

```shell
# 卸载逻辑卷
# umount /dev/testvg/testlv

# 删除逻辑卷
# lvremove /dev/testvg/testlv
```
