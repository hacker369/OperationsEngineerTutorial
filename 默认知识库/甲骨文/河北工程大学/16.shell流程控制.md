# 16.shell流程控制

# 一、条件判断

我们经常会判断一些变量或者表达式的结果是真还是假，这时候就需要使用到条件判断。

## test进行条件判断

`test 表达式`判断表达式的结果，返回0表示表达式成立，返回非0表示表达式不成立！

```shell
# 定义一个变量name
[root@localhost ~]# name=zhangsan

# 使用变量name
[root@localhost ~]# echo $name
zhangsan

# 判断name变量的值是否等于zhangsan，使用=号比较，注意等号两侧有空格
[root@localhost ~]# test $name = zhangsan

# 输出上一条命令返回的结果，如果是0表示正确，如果非0表示不正确
[root@localhost ~]# echo $?
0

# 判断name变量的值是否等于zhangsan，使用=号比较，注意等号两侧有空格
[root@localhost ~]# test $name = haha

# 输出上一条命令返回的结果，如果是0表示正确，如果非0表示不正确
[root@localhost ~]# echo $?
1
```

## \[ ]进行条件判断

`[ 条件表达式 ]`判断表达式的结果，返回0表示表达式成立，返回非0表示表达式不成立！注意中括号内部两边的空格。

```shell
# 定义变量
[root@localhost ~]# name=lisi

# 判断表达式的结果
[root@localhost ~]# [ $name = lisi ]

# 输出上一条命令返回的结果，如果是0表示正确，如果非0表示不正确
[root@localhost ~]# echo $?
0

# 判断表达式的结果
[root@localhost ~]# [ $name = wangwu ]

# 输出上一条命令返回的结果，如果是0表示正确，如果非0表示不正确
[root@localhost ~]# echo $?
1

# 判断任意一个非空字符串，会认为是正确的
[root@localhost ~]# [ aaaaaaaaaa ]
[root@localhost ~]# echo $?
0

# 空内容，会认为是错误的
[root@localhost ~]# [    ]
[root@localhost ~]# echo $?
1

# 定义变量
[root@localhost ~]# name=liubei

# 判断表达式的结果 != 表示不等于
[root@localhost ~]# [ $name != hello ]

# 输出上一条命令返回的结果，如果是0表示正确，如果非0表示不正确
[root@localhost ~]# echo $?
0

# 判断表达式的结果 != 表示不等于
[root@localhost ~]# [ $name != liubei ]

# 输出上一条命令返回的结果，如果是0表示正确，如果非0表示不正确
[root@localhost ~]# echo $?
1
```

## 常用判断条件

在上面的条件判断中，我们都是判断的字符串之间的相等与不等。如果要判断其它的内容，就不能直接使用 = 或者 != 进行判断了。

**两个整数之间比较**

| 符号 | 说明 |
| --- | --- |
| -eq | 等于（equal） |
| -ne | 不等于（not equal） |
| -lt | 小于（less than） |
| -le | 小于等于（less equal） |
| -gt | 大于（greater than） |
| -ge | 大于等于 |

```shell
# 判断2小于5吗
[root@localhost ~]# [ 2 -lt 5 ]
[root@localhost ~]# echo $?
0

# 判断2大于5吗
[root@localhost ~]# [ 2 -gt 5 ]
[root@localhost ~]# echo $?
1
```

***

**按照文件权限进行判断**

| 符号 | 说明 |
| --- | --- |
| -r | 有读的权限（read） |
| -w | 有写的权限（write） |
| -x | 有执行的权限（execute） |

```shell
[root@localhost ~]# touch abc.doc

[root@localhost ~]# ls -l abc.doc
-rw-r--r-- 1 root root 0 Mar  5 15:46 abc.doc

# 判断文件是否有读的权限
[root@localhost ~]# [ -r abc.doc ]
[root@localhost ~]# echo $?
0

# 判断文件是否有写的权限
[root@localhost ~]# [ -w abc.doc ]
[root@localhost ~]# echo $?
0

# 判断文件是否有执行的权限
[root@localhost ~]# [ -x abc.doc ]
[root@localhost ~]# echo $?
1
```

***

**按照文件类型进行判断**

| 符号 | 说明 |
| --- | --- |
| -e | 文件存在（existence） |
| -f | 文件存在并且是一个常规的文件（file） |
| -d | 文件存在并且是一个目录（directory） |

```shell
# 判断一个文件是否存在
[root@localhost ~]# [ -e abc.doc ]
[root@localhost ~]# echo $?
0

# 判断一个文件是否存在
[root@localhost ~]# [ -e xyz.txt ]
[root@localhost ~]# echo $?
1

# 判断文件存在并且是一个常规的文件
[root@localhost ~]# [ -f test1.sh ]
[root@localhost ~]# echo $?
0

# 判断文件存在并且是一个常规的文件
[root@localhost ~]# [ -f /root ]
[root@localhost ~]# echo $?
1

# 判断文件存在并且是一个目录
[root@localhost ~]# [ -d test1.sh ]
[root@localhost ~]# echo $?
1

# 判断文件存在并且是一个目录
[root@localhost ~]# [ -d /root/ ]
[root@localhost ~]# echo $?
0
```

## 多条件判断

前面的操作中，我们都是一个判断。实际中，我们可能也需要判断多个条件。

`&&`表示短路与，前面一条命令执行成功时，才会执行后一条命令

`||`表示短路或，前面一条命令执行失败时，才会执行后一条命令

```shell
[root@localhost ~]# num=6
[root@localhost ~]# [ $num -lt 10 ] && echo "$num < 10" || echo "$num >= 10"
6 < 10

[root@localhost ~]# num=15
[root@localhost ~]# [ $num -lt 10 ] && echo "$num < 10" || echo "$num >= 10"
15 >= 10
```

# 二、流程判断

## if判断

### 单分支

```shell
if [ 条件判断 ]; then	=> 因为一行有多个命令或关键字，可以使用分号;分隔开
  程序
fi

或者
if [ 条件判断 ]
then
  程序
fi


注意：条件判断的[]内部两侧要有空格；if后面要有空格，然后再写[]
```

案例：在终端中简单使用if进行条件判断

```shell
[root@localhost ~]# age=20

# 因为我们一行写了多个命令关键字，需要写完一部分后然后用;隔开
[root@localhost ~]# if [ $age -gt 18 ];then echo "成年了";fi
成年了

[root@localhost ~]# age=15

[root@localhost ~]# if [ $age -gt 18 ];then echo "成年了";fi
[root@localhost ~]#
```

案例：编写shell脚本，获取一个字符串参数，使用if进行条件判断

```shell
[root@localhost ~]# touch test1.sh

[root@localhost ~]# vim test1.sh
#!/bin/bash
if [ $1 = "zhangsan" ]		=> 判断第一个参数是否等于zhangsan
then
        echo "hello,zhangsan"
fi

[root@localhost ~]# chmod +x test1.sh

[root@localhost ~]# ./test1.sh xiaoming

[root@localhost ~]# ./test1.sh zhangsan
hello,zhangsan

问题:
上面的脚本其实是有点问题的，如果用户执行脚本的时候输入参数的话，那没问题；如果用户在执行脚本的时候没有输入参数，那$1得到的就是空，空和zhangsan比较就会报错！
[root@localhost ~]# ./test1.sh
./test1.sh: line 2: [: =: unary operator expected

可以优化一下脚本内容
[root@localhost ~]# vim test1.sh
#!/bin/bash
在变量$1的后面拼接字符串x，在zhangsan后面拼接字符串x，这样即使$1为空也不会影响判断
if [ "$1"x = "zhangsan"x ]
then
        echo "hello,zhangsan"
fi

[root@localhost ~]# ./test1.sh
[root@localhost ~]# ./test1.sh abc
[root@localhost ~]# ./test1.sh zhangsan
hello,zhangsan
```

案例：编写shell脚本，获取参数，然后使用if进行多条件的判断

```shell
[root@localhost ~]# touch test2.sh

[root@localhost ~]# vim test2.sh
#!/bin/bash
if [ $1 -ge 18 ] && [ $1 -lt 35 ]
then
        echo "可以来上班"
fi

[root@localhost ~]# chmod +x test2.sh

[root@localhost ~]# ./test2.sh 12
[root@localhost ~]# ./test2.sh 25
可以来上班
[root@localhost ~]# ./test2.sh 45
[root@localhost ~]#
```

案例：改造上面的案例，上面需要在if后面写两个`[ ]`进行条件判断，其实是可以合成一个的！

```shell
[root@localhost ~]# touch test3.sh

[root@localhost ~]# vim test3.sh
#!/bin/bash
if [ $1 -ge 18 -a $1 -lt 35 ]
then
        echo "你可以上班了"
fi

[root@localhost ~]# chmod +x test3.sh

[root@localhost ~]# ./test3.sh 15
[root@localhost ~]# ./test3.sh 26
你可以上班了
[root@localhost ~]# ./test3.sh 43
[root@localhost ~]#
```

上面的`[ $1 -ge 18 -a $1 -lt 35 ]`中是将两个判断的条件写到一个`[ ]`中了，其中`-a`表示and，代替了`&&`；如果是`||`的话，我们使用`-o`也就是or表示或的意思。

## case语句

使用if语句可以进行条件判断，如果分支多了，需要写很多if、elif等语句，且不美观不利于阅读。我们可以使用case语句进行多分支的判断。不过case语句只能判断是否等于某个值。

```shell
case $变量名 in
"值1")
  如果变量的值等于值1，则执行程序1
;;
"值2")
  如果变量的值等于值2，则执行程序2
;;
...省略其他的分支
*)
  如果变量的值都不是以上的值，则执行此程序
;;
esac

注意事项：
1. case行尾必须为单词"in"，每个模式匹配必须以右括号")"结束
2. 双分号";;"表示命令序列结束，相当于Java中的break
3. 最后的"*)"表示默认模式，相当于Java中的default
```

```shell
[root@localhost ~]# touch test6.sh

[root@localhost ~]# vim test6.sh
#!/bin/bash

case $1 in
1)
        echo "one"
;;
2)
        echo "two"
;;
3)
        echo "three"
;;
*)
        echo "other"
esac

[root@localhost ~]# chmod +x test6.sh

[root@localhost ~]# ./test6.sh 1
one

[root@localhost ~]# ./test6.sh 2
two

[root@localhost ~]# ./test6.sh 3
three

[root@localhost ~]# ./test6.sh 6
other
```

## for循环

### 第一种方式

```shell
for (( 初始值;循环控制条件;变量变化 ))
do
  程序
done
```

```shell
[root@localhost ~]# touch test1.sh

[root@localhost ~]# vim test1.sh
#!/bin/bash
之前我们说比较数字使用le、eq等，比较字符串使用=、!=等，这里之所以能用<=是因为我们把判断条件这些写在(())里面了，在(())里面是可以进行数学上的一些运算、比较的
for (( i=0;i<=$1;i++ ))
do
        sum=$[ $sum+$i ]
done
echo "总和是：$sum"

[root@localhost ~]# chmod +x test1.sh

[root@localhost ~]# ./test1.sh 100
总和是：5050

[root@localhost ~]# ./test1.sh 10
总和是：55
```

### 第二种方式

```shell
for 变量 in 值1 值2 值3...
do
  程序

```

案例1：入门案例

```shell
[root@localhost ~]# vim test.sh
#!/bin/bash
for os in windows linux mac

        echo $os

[root@localhost ~]# ./test.sh
windows
linux
mac
```

案例3：我们知道`$*`是可以获取所有的参数的，但获取到的是一个整体；`$@`也可以获取所有的参数，但获取到的其实是一个集合。

```shell
[root@localhost ~]# touch test.sh

[root@localhost ~]# vim test.sh
#!/bin/bash
echo '--------$*----------'  => 注意这里用了单引号，则里面的$*会原模原样输出
for i in $*		=> $* 没有用引号引起来的话，它也可以可以获取所有参数，并且是集合的效果
do
        echo $i
done

echo '--------$@----------'
for i in $@
do
        echo $i
done

[root@localhost ~]# chmod +x test.sh

[root@localhost ~]# ./test.sh a b c
--------$*----------
a
b
c
--------$@----------
a
b
c
```

```shell
修改上面的脚本内容为
#!/bin/bash
echo '--------$*----------'
for i in "$*"  => $*通过双引号引起来，则获取到的所有参数是一个整体了！
do
        echo $i
done

echo '--------$@----------'
for i in "$@"
do
        echo $i
done

[root@localhost ~]# ./test.sh a b c

--------$*----------
a b c
--------$@----------
a
b
c
```

## while循环

```shell
while [ 条件判断 ]
do
  程序
done
```

```shell
[root@localhost ~]# touch test.sh

[root@localhost ~]# vim test.sh
#!/bin/bash
i=1
while [ $i -le $1 ]
do
        sum=$[ $sum+$i ]
        i=$[ $i+1 ]
done
echo "结果是：$sum"

[root@localhost ~]# chmod +x test.sh

[root@localhost ~]# ./test.sh 10
结果是：55

[root@localhost ~]# ./test.sh 100
结果是：5050
```

# 三、read读取控制台输入

```shell
read [选项] [参数]
选项说明：
-p：指定读取值时的提示符  prompt
-t：指定读取值时等待的时间(秒)，如果-t不写表示一直等待
参数说明：
变量：指定读取值的变量名
```

```shell
[root@localhost ~]# touch test.sh

[root@localhost ~]# vim test.sh
#!/bin/bash
read -t 10 -p "请输入你的姓名：" name
echo "你好,$name"

[root@localhost ~]# chmod +x test.sh

[root@localhost ~]# ./test.sh
请输入你的姓名：张三
你好,张三
```
